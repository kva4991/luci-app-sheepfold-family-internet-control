# Резервное копирование и импорт настроек

<!-- §cfgbak1 -->

## Что сохраняется

Экспорт из `Настройки → Разное` сохраняет:

- все UCI-секции Sheepfold, включая устройства, группы, расписания, администраторов, списки устройств, сайты, мессенджер, интеграции и настройки программы;
- постоянные аренды DHCP;
- UCI-секции Wi-Fi-радиомодулей и сетей.

В копию намеренно не входят root-пароль OpenWrt, активные LuCI-сессии, журналы, временные DHCP-аренды и Bearer-токены уже сопряжённых Android-устройств. После переноса на другой роутер родительские телефоны следует сопрячь заново.

## Читаемый JSON

Формат `sheepfold-settings-export-v2` предназначен для проверки человеком и не содержит секретов. Пароли, API-ключи, bot tokens, cookies и session values заменяются точным маркером `[secret]`.

При импорте на тот же роутер Sheepfold сохраняет совпадающий текущий секрет из одноимённой UCI-секции. Если секции или секрета на роутере нет, поле остаётся ненастроенным. Обычный JSON с настоящим значением в секретном поле отклоняется: файл не может притвориться безопасным экспортом.

## Зашифрованная полная копия

Режим `Зашифрованный полный бэкап` включает пароли и токены из экспортируемых UCI-конфигов. Перед скачиванием браузер шифрует JSON локально:

- AES-256-GCM;
- ключ из пользовательского пароля через PBKDF2-HMAC-SHA-256;
- 250 000 итераций;
- случайная salt 16 байт;
- случайный IV 12 байт.

Пароль не отправляется на роутер и не сохраняется. Восстановить копию без него нельзя. Открытый JSON с `containsSecrets=true` импортировать запрещено; полный набор секретов принимается только после успешной расшифровки контейнера `sheepfold-settings-encrypted-v1`.

## Безопасное применение

До записи LuCI проверяет формат, размер, имена секций и опций, допустимые типы DHCP/Wi-Fi-секций, обязательные разделы Sheepfold и отсутствие одного MAC одновременно в белом и чёрном списках устройств. Максимальный размер выбранного файла — 5 МБ.

Импорт заменяет:

- все секции `/etc/config/sheepfold`;
- только секции типа `host` в `/etc/config/dhcp`;
- только `wifi-device` и `wifi-iface` в `/etc/config/wireless`.

Посторонние типы секций DHCP и wireless не удаляются. Перед применением создаётся снимок текущих значений в памяти LuCI; при ошибке сохранения выполняется попытка отката через тот же UCI apply. После успеха Sheepfold сразу синхронизирует firewall и страницу блокировки, затем с небольшой задержкой перезагружает dnsmasq и Wi-Fi, чтобы ответ LuCI успел вернуться без ложного XHR timeout.

## Что проверить на живом роутере

1. Обычный JSON на том же роутере сохраняет Telegram-токен и пароль Wi-Fi.
2. После импорта обычного JSON на чистый роутер отсутствующие секреты остаются пустыми.
3. Зашифрованная копия восстанавливает Wi-Fi, мессенджер и клиентов только с правильным паролем.
4. После восстановления применяются чёрный и белый списки устройств, расписания и глобальное состояние интернета.
5. Wi-Fi и dnsmasq перезапускаются после ответа LuCI, а страница снова открывается после переподключения.
6. Повреждённый файл и конфликт списков не меняют текущий конфиг.
