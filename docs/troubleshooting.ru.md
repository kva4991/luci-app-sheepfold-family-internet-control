# Sheepfold: справочник ошибок и их исправления

<!-- §trouble -->

Этот документ предназначен прежде всего для разработчиков и AI-агентов, но команды и объяснения подходят и пользователю, который собирает Sheepfold на Windows или устанавливает пакет на OpenWRT-роутер.

Здесь собраны только уже встречавшиеся и понятные нам классы ошибок. Не исправляйте незнакомую ошибку по похожему тексту вслепую: сначала сохраните полный вывод команды, определите слой проблемы и запустите ближайшую проверку.

## Быстрая диагностика

Из корня репозитория на Windows:

```powershell
git status --short --branch
powershell -ExecutionPolicy Bypass -File tools\windows\check.ps1
node --test tests\windowsToolchain.test.mjs
git diff --check
```

Полный тестовый прогон перед публикацией:

```powershell
npm.cmd test
```

Проверять весь проект после каждой строки не нужно. Во время разработки запускайте ближайший целевой тест, а полный набор — один раз перед push или после изменения общего API/UCI/backend-контракта.

## Как читать ошибку

1. Зафиксируйте точную команду, exit code и полный текст, а не только всплывающее сообщение LuCI.
2. Определите слой: Windows, тест, Android, IPK/opkg, UCI/backend, LuCI, GitHub или сеть.
3. Проверьте состояние после операции. Сообщение «сохранено» ещё не доказывает, что UCI действительно записан.
4. Исправьте причину и добавьте регрессионный тест, если ошибка воспроизводима кодом.
5. Если находка неочевидная, обновите этот справочник, [`agent-gotchas.ru.md`](agent-gotchas.ru.md) и карту [`§-тегов`](dev/tag-map.md).

## Windows и инструменты

### Команда установлена, но не находится

**Симптом:** `python`, `java`, `adb`, `sdkmanager`, `rg` или `winget` «не распознано как имя команды» сразу после установки.

**Причина:** открытый PowerShell или процесс Codex не перечитал пользовательский и системный `PATH`.

**Исправление:** откройте новый PowerShell/Codex. Для продолжения в текущем процессе один раз выполните:

```powershell
$env:Path = ([Environment]::GetEnvironmentVariable('Path','Machine') + ';' + [Environment]::GetEnvironmentVariable('Path','User'))
```

Затем запустите `tools\windows\check.ps1`. Не устанавливайте вторую копию программы только из-за старого `PATH`.

### Python открывает Microsoft Store или не найден

**Причина:** срабатывает Windows App Execution Alias либо каталог настоящего Python отсутствует в `PATH`.

**Исправление:** используйте проектный установщик. Он устанавливает Python через `winget`, находит реальный `python.exe` и добавляет только одну наиболее новую установку в `PATH`.

### `JAVA_HOME is not set`

**Проверка:**

```powershell
java -version
echo $env:JAVA_HOME
```

Sheepfold требует JDK 17, а не только JRE и не произвольную новую JDK. Полный `setup.ps1` находит Temurin JDK 17, задаёт `JAVA_HOME` и проверяет наличие `java.exe` и `javac.exe`.

### `Cannot overwrite variable Home because it is read-only or constant`

**Причина:** PowerShell нечувствителен к регистру; параметр `$Home` конфликтует со встроенной read-only переменной `$HOME`.

**Исправление для разработчика:** переименовать параметр, например в `$JavaHomePath`. В Sheepfold это закреплено тестом `windowsToolchain.test.mjs`; не возвращайте имя `$Home` в параметры функций.

### Ошибка `Expand-Archive` с внутренним `Remove-Item`

**Причина:** Windows PowerShell 5 нестабильно распаковывает официальный архив Android command-line tools, особенно из длинного пути.

**Исправление:** проектный установщик использует `.NET ZipFile` и короткий временный каталог. Не заменяйте этот код на `Expand-Archive`. Контракт: `§zipps51`.

### `npm.ps1 cannot be loaded because running scripts is disabled`

Используйте исполняемый wrapper:

```powershell
npm.cmd test
```

Менять общую Execution Policy Windows ради запуска тестов проекта не требуется.

### PowerShell-скрипт показывает `Unexpected token` внутри русского текста

**Причина:** Windows PowerShell 5.1 открыл UTF-8 файл без BOM как старую ANSI-кодировку. Новый PowerShell может разобрать тот же файл без ошибки, поэтому проверка только текущим shell недостаточна.

**Исправление:** хранить Windows `.ps1` с русским текстом в UTF-8 BOM и проверять его именно `powershell.exe` 5.1. Для `tools/router-testing/*.ps1` это закреплено тестом `liveRouterHarness.test.mjs`. (§routerharness)

### `ssh-keygen: Too many arguments` при пустой passphrase

**Причина:** Windows PowerShell 5.1 удаляет настоящий пустой native-аргумент в `-N ''`, и `ssh-keygen` получает смещённый список параметров.

**Исправление для скрипта:** передавать `-N '""'`; Windows native parser превращает эту строку именно в пустую passphrase. Не переходить ради обхода на пароль в коде или интерактивное ожидание, которое зависнет у агента.

### `Export-ModuleMember can only be called from inside a module`

**Причина:** общий helper имеет расширение `.ps1`, но загружается как настоящий `.psm1`-модуль.

**Исправление:** для простого общего `.ps1` использовать dot-source: `. (Join-Path $PSScriptRoot 'helper.ps1')`. Либо осознанно преобразовать его в `.psm1`; не смешивать два механизма. (§routerharness)

## Android SDK и Gradle

### `gradle` не распознано

Глобальный Gradle Sheepfold не нужен. Используйте сохранённый в репозитории Gradle Wrapper:

```powershell
android\gradlew.bat -p android assembleDebug --stacktrace
android-child\gradlew.bat -p android-child assembleDebug --stacktrace
```

### Не скачивается Build Tools или Platform

**Симптом:** `Failed to install build-tools;...`, timeout или неполный каталог SDK.

**Исправление:** повторите полную команду `setup.ps1`. Она делает до пяти попыток, удаляет только неполный компонент и сохраняет уже исправные SDK-пакеты. Не удаляйте целиком `%LOCALAPPDATA%\Android\Sdk` из-за сбоя одного файла.

### Предупреждение о несовпадении SDK XML

Сначала проверьте, заканчивается ли сборка ошибкой. Само предупреждение о версии SDK XML не всегда ломает сборку. Обновляйте command-line tools через проектный установщик, но не меняйте `compileSdk` или Android Gradle Plugin только ради исчезновения предупреждения.

### Gradle долго ничего не выводит

Первый запуск может скачивать официальный Gradle distribution и зависимости. Посмотрите URL и каталог `%USERPROFILE%\.gradle\wrapper\dists`, дождитесь завершения или сетевого timeout. `setup.ps1` заранее прогревает wrapper для обоих Android-проектов.

## Android-приложение и сопряжение

### Play Protect подозревает тестовый APK

Debug APK установлен не из магазина и не подписан будущим production-ключом, поэтому Play Protect может показать предупреждение о неизвестном разработчике. Это не доказывает наличие вредоносного кода, но и не должно скрываться от пользователя. Для публичного релиза нужны стабильная подпись, зафиксированный SHA-256, публикационная политика и проверяемый источник загрузки. Не советуйте отключать Play Protect целиком.

### QR виден в LuCI, но телефон его не распознаёт

Проверьте разрешение камеры, квадратную область preview, контраст и размер QR, а также возможность выбрать изображение из галереи. Затем декодируйте тот же QR независимым тестом и проверьте схему payload. Не считайте проблемой камеры ситуацию, когда QR декодируется, но сервер отклоняет уже использованный одноразовый token.

### QR распознан, но появляется ошибка подключения

Проверяйте по порядку: телефон находится в локальной Wi-Fi/Ethernet-сети роутера; `https://<router-ip>/.well-known/sheepfold.json` сообщает актуальный порт; HTTPS endpoint `https://<router-ip>:<порт>/cgi-bin/sheepfold-api` отвечает JSON с `service=sheepfold`; одноразовый код ещё жив и не использован; backend видит IP телефона в DHCP leases, `ip neigh` или ARP и привязал его к администратору. Смотрите router log и HTTP status. Не добавляйте автоматический fallback на HTTP: оба Android-приложения должны оставаться HTTPS-only. Пользовательская ошибка не должна быть пустой или равной `undefined`; сохраняйте конкретную безопасную причину этапа подключения (§dscqr01).

### Телефон получил token, но не появился у администратора

Успешное сопряжение должно быть атомарной backend-операцией: сжечь token, создать/обновить устройство с числовым ID, записать связь с администратором, удалить обычные группы/расписания устройства, добавить его в allowlist и вернуть подтверждённое состояние. Если UI показывает успех раньше commit, после обновления связь исчезнет. Повторное использование того же token обязано отклоняться.

## Тесты

### `status: null`, `EPERM` или тест не создал дочерний процесс

**Причина:** песочница Codex запретила дочернюю сборку, хотя проверяемый код может быть исправен.

**Исправление:** повторите только упавший тест с разрешённым системным запуском. Не переписывайте production-код по одному `status: null`. Для IPK обычно достаточно:

```powershell
node --test tests\testIpkI18n.test.mjs tests\testIpkPermissions.test.mjs
```

### Полный набор кажется зависшим

Тесты безопасного автоопределения устройств выполняют реальные временные сценарии и заметно длиннее остальных. Отсутствие вывода десятки секунд не означает зависание. Дождитесь итоговых `pass`, `fail` и exit code; не запускайте второй полный прогон параллельно.

### Тест версии пакета падает после правки

Проверьте единый номер в `Makefile`, тесте asset versioning, документации текущей сборки и имени ожидаемого IPK. После изменения LuCI JS/CSS поднимайте `PKG_RELEASE`; версия ассетов должна измениться вместе с ним.

### Тест прошёл локально, GitHub Actions красный

Откройте конкретный job и первую настоящую ошибку, а не только общую плашку. Проверяйте и `push`, и `pull_request` jobs, но не считайте их двумя разными дефектами, если traceback одинаковый. Полезные команды:

```powershell
gh run list
gh run view <run-id> --log-failed
```

## Сборка и установка IPK

### `Malformed package file /tmp/upload.ipk`

**Причина:** пакет собран не в формате, который ожидает OpenWRT `opkg`, повреждён при упаковке или загрузке.

**Исправление:** собирайте только проектным скриптом:

```powershell
python scripts\build-test-ipk.py
```

Правильный тестовый IPK Sheepfold — gzip-сжатый tar с `debian-binary`, `control.tar.gz` и `data.tar.gz`. Не собирайте его 7-Zip, Windows `tar` или самодельным `ar`.

### `postinst: Permission denied`

Проверьте Unix-права `postinst` и файлов `usr/libexec/sheepfold/*` внутри архива. Они должны быть исполняемыми (`0755`), а `postinst` дополнительно восстанавливает права helpers после установки. Регрессионный тест: `testIpkPermissions.test.mjs`.

### `Missing new line character at end of file`

Проверьте завершающий LF в control-файлах и shell-скриптах пакета. Не исправляйте это переводом всех shell-файлов в CRLF: на роутере нужен Unix LF.

### `cannot find dependency jsonfilter`

Это зависимость OpenWRT, а не файл интерфейса Sheepfold. Убедитесь, что репозитории пакетов роутера соответствуют установленной прошивке, обновите индекс пакетов и установите зависимость. Не вшивайте случайный бинарник другой архитектуры в `all.ipk`.

### `Existing conffile /etc/config/sheepfold is different`

Это обычно не ошибка установки. `opkg` сохранил пользовательский конфиг и положил новый шаблон рядом как `/etc/config/sheepfold-opkg`. Миграцию значений должен выполнять `postinst`/UCI-defaults без удаления пользовательских списков. Не заменяйте рабочий конфиг новым файлом вслепую.

### Пакет сообщает `is up to date`

Поднимите `PKG_RELEASE`, пересоберите пакет и проверьте имя файла. Повторная установка того же номера не доказывает, что новый код попал на роутер.

## UCI и backend роутера

### `uci: Entry not found` для `uci show sheepfold`

Конфиг `/etc/config/sheepfold` отсутствует или не был восстановлен после неудачной установки. Установка должна создать полный валидный конфиг со всеми обязательными секциями и значениями по умолчанию, а не только пустой файл. После восстановления проверьте:

```sh
uci show sheepfold
uci commit sheepfold
```

### `RPC call to uci/get failed with ubus code 9`

Частые причины: отсутствует конфиг/секция, ACL не разрешает чтение или LuCI обращается к неверному имени UCI config. Сначала проверьте `/etc/config/sheepfold`, `uci show sheepfold`, ACL JSON и системный log. Не маскируйте ошибку подстановкой тестовых данных во frontend.

### UI пишет «сохранено», но после обновления снова старое значение

Проверьте всю цепочку: правильная именованная UCI-секция → `uci set`/LuCI UCI API → `save` → `commit` → при необходимости `apply` → повторное чтение backend. Особое внимание именам `messenger_global`, `pairing_global` и типам секций. Всплывающее уведомление показывайте только после подтверждённого успешного чтения сохранённого значения.

### LuCI показывает «Не принятые изменения»

После явной кнопки Sheepfold «Сохранить» frontend должен не только изменить локальное UCI-состояние, но и применить/принять очередь изменений LuCI. Не включайте autosave на blur или checkbox: пользовательская модель остаётся одной осознанной кнопкой «Сохранить».

### Временный доступ стал постоянным

Backend должен помнить, добавлял ли он MAC во временный allowlist, и удалить только это временное добавление после срока. `blocklist` и `status=blocked` всегда сильнее временного доступа и WPS. Проверки находятся в `deviceTempAccessUi.test.mjs` и `routerBackendAccessRules.test.mjs`.

## LuCI и браузер

### После обновления виден старый JS/CSS

Не советуйте пользователю постоянно чистить кэш. Поднимите `PKG_RELEASE` и синхронный `ui_asset_version`; LuCI должен загружать ассеты с новой общей query-версией. См. [`luci-cache.ru.md`](luci-cache.ru.md).

### `factory yields invalid constructor`

LuCI view wrapper вернул объект или применил неподходящий способ наследования вместо допустимого view constructor. Не используйте устаревший `BaseView.extend` для текущей обёртки Sheepfold. После исправления запустите `secureOverviewWrapper.test.mjs` и проверьте страницу на роутере.

### Иконки отсутствуют

Не подключайте внешний CDN: роутер может быть без интернета, а браузер может блокировать ресурс. Используйте встроенные LuCI-ресурсы либо локальные SVG/CSS-иконки пакета с тем же asset versioning.

## Git и GitHub

### `git push` ждёт вход и завершается timeout

Git Credential Manager мог открыть невидимый интерактивный вход. Проверьте `gh auth status`, выполните явную авторизацию нужного личного аккаунта, затем повторите push. Перед push обязательно убедитесь, что `git status`, текущая ветка и remote относятся к Sheepfold.

### Правки находятся в архиве с `.git`

Распакуйте архив в отдельный каталог, сравните HEAD и рабочее дерево с текущим репозиторием, переносите только содержательные файлы. Не копируйте архивные `.git`, `.build`, кэши и артефакты поверх рабочего репозитория.

### В рабочем дереве есть чужие изменения

Не выполняйте `git reset --hard`, `git checkout --` или массовое удаление. Отделите файлы задачи, изучите пересечения и работайте с существующими правками. Если пересечение делает задачу неоднозначной, остановитесь и объясните пользователю конкретный конфликт.

## Сеть, Telegram и updater

### Telegram сообщает об успешном тесте, но сообщение не пришло

Проверьте сохранённые token/user ID повторным чтением UCI, ответ Telegram API и разрешённый ID пользователя. Не считайте HTTP-запрос успешным только потому, что frontend отправил его. Токены никогда не выводите в журнал.

### Updater получает `XHR request timed out`

Долгую загрузку и установку нельзя держать одним синхронным XHR LuCI. Кнопка должна создать задачу, модалка — показывать проверку/установку, а frontend — опрашивать короткий status endpoint. Отдельно различайте «обновлений нет», сетевую ошибку GitHub и ошибку установки IPK.

### Проверка интернета ошибочно сообщает «восстановлен» при первом запуске

Первое успешное измерение — инициализация состояния, а не восстановление. Событие «Интернет восстановлен» пишется только после ранее зафиксированного перехода в offline/limited. Для России не полагайтесь только на Google/Cloudflare; используйте страновую диагностическую цепочку из `backend-design.ru.md`.

## Чего не делать

- Не удалять весь Android SDK или Gradle cache из-за одного timeout.
- Не ставить глобальный Gradle вместо project wrapper.
- Не собирать IPK архиватором общего назначения.
- Не чинить код по одному `status: null` из песочницы.
- Не подменять отсутствующий backend статическими «боевыми» значениями во frontend.
- Не показывать «сохранено» до commit и контрольного чтения.
- Не обходить blocklist временным доступом, WPS, группой или расписанием.
- Не очищать пользовательский `/etc/config/sheepfold` при обновлении пакета.
- Не пушить SDK, APK, IPK, `.gradle`, `.build` и скачанные архивы.
- Не скрывать точную первичную ошибку за общим сообщением «не удалось» в техническом журнале; пользовательский текст может быть простым, но диагностический вывод должен сохранять безопасную причину без секретов.

## Куда добавлять новую ошибку

Добавляйте её в подходящий раздел этого файла в формате «симптом → причина → проверка → исправление». Если ошибка относится к архитектурному контракту, поставьте существующий §-тег рядом с кодом и обновите [`docs/dev/tag-map.md`](dev/tag-map.md). Тест должен воспроизводить именно причину ошибки, а не только искать случайную строку сообщения.
