# Два варианта роутерного пакета Sheepfold

<!-- §prodvar -->

## Принятое решение

Из одного репозитория и общего кода выпускаются две редакции роутерного пакета:

1. **Sheepfold** — семейное управление интернетом без AI-backend и без подробного сбора активности устройств.
2. **Sheepfold - AI Support** — тот же контроль плюс ИИ-помощник, proxy на роутере, системные промпты и явно включаемый контекст.

Android не делится на Standard/AI Support. Выпускаются одно родительское и одно детское приложение. Оба содержат небольшой AI-клиент, но скрывают все связанные вкладки и настройки, пока подключённый роутер не вернёт подтверждённую возможность ИИ.

Для OpenWrt 24.10 каждая редакция выпускается как IPK, а для OpenWrt 25.12 —
как настоящий пакет apk-tools v3. OpenWrt APK нельзя получать переименованием
или перепаковкой IPK (§owrtci1).

## Релизные артефакты

Полный релиз содержит четыре роутерных пакета и два Android-приложения:

```text
luci-app-sheepfold-family-internet-control_<version>_all.ipk
luci-app-sheepfold-ai-support_<version>_all.ipk
luci-app-sheepfold-family-internet-control-<version>.apk  # OpenWrt apk-tools v3
luci-app-sheepfold-ai-support-<version>.apk               # OpenWrt apk-tools v3
sheepfold-parent-v<version>.apk
sheepfold-child-v<version>.apk
```

Четыре роутерных файла являются разными форматами/редакциями одного
установленного пакета `luci-app-sheepfold-family-internet-control`. На роутере
одновременно активна только одна редакция. Standard можно заменить на AI
Support как обычное обновление без удаления UCI-конфига и повторной настройки
семьи. Разные имена release-файлов нужны человеку и updater, чтобы выбрать
правильные редакцию и формат.

Старое тестовое внутреннее имя `luci-app-sheepfold-ai-support` объявлено заменяемым. Это позволяет перейти и с раннего AI-пакета на новую общую модель пакета.

При смене редакции на той же версии установочный скрипт использует штатную
принудительную переустановку через единый package-manager adapter, предварительно
сохранив `/etc/config/sheepfold`. На 24.10 adapter вызывает `opkg`, на 25.12 —
`apk` v3. При переходе на более новую версию пакетный менеджер видит обновление
той же установленной сущности (§pkgmgr1).

Если владелец вручную загружает через стандартную страницу LuCI другую редакцию
с тем же номером версии, системный менеджер вправе ответить `up to date`. Для
такого перехода нужен установочный скрипт: он сравнивает редакцию/версию и
безопасно выбирает принудительную переустановку.

Android имеет стабильные идентификаторы:

```text
app.sheepfold.android
app.sheepfold.child
```

## Граница роутерного пакета

Обычный Standard IPK/OpenWrt APK физически не содержит:

- AI endpoint и handler;
- системные промпты;
- настройки ключей провайдеров;
- подробный сбор активности для анализа ИИ;
- AI-раздел LuCI.

AI Support содержит общий контроль и эти дополнительные модули. Ключи DeepSeek, Gemini или Grok хранятся только на роутере. Обычный административный журнал остаётся в обеих редакциях и обоих форматах роутерного пакета.

При переходе из AI Support в Standard AI-поля UCI не исполняются, но не стираются автоматически. Поэтому обратный переход не требует повторно вводить параметры провайдера. Удаление сохранённых AI-ключей должно быть отдельным явным действием владельца.

Исходный payload использует маркеры `SHEEPFOLD_AI_BEGIN/END` и
`SHEEPFOLD_STANDARD_BEGIN/END`. Единый модуль
`scripts/sheepfold_variants.py` удаляет противоположные блоки до передачи feed
официальному SDK или тестовому IPK-сборщику; маркеры запрещены в готовом пакете.

## Server-driven Android

В Android нет product flavors и копий навигации:

```text
android/app/src/main/          # единый родительский APK
android-child/app/src/main/    # единый детский APK
```

Родительское приложение показывает вкладку ИИ только при `capabilities.aiAssistant=true` в авторизованном ответе `/router-info`.

Роутер возвращает `true`, только когда одновременно:

- установлена редакция AI Support и присутствует AI handler;
- ИИ включён в настройках;
- выбран поддерживаемый провайдер;
- для выбранного провайдера сохранён непустой API-ключ.

Обычный IPK всегда возвращает `false`. Недоступность роутера, старый ответ без поля или ошибка разбора также трактуются как `false`.

Детское приложение дополнительно требует `childAiAvailable=true` из `/client-status`. Этот флаг учитывает разрешение родителя. Если ИИ доступен, но устройство ещё не находится в личной группе, вкладка видна с баннером, а отправка сообщения заблокирована. Финальное разрешение backend проверяет повторно при каждом запросе.

Capability не заменяет авторизацию и не считается доказательством доступности внешнего провайдера в эту секунду. Он подтверждает, что router-side proxy установлен и настроен; сетевую ошибку провайдера приложение показывает только после запроса.

## Общая часть

- устройства, числовые ID, типы и автоопределение;
- белый/чёрный списки устройств, группы, расписания и временный доступ;
- глобальное управление интернетом и аварийно-полезные сайты;
- firewall4/nftables и совместимость с AdGuard Home/Podkop;
- Wi-Fi, WPS, LED, администраторы, QR-сопряжение и HTTPS API;
- Telegram/VK, обновление, импорт, экспорт и резервное копирование;
- административный и технический журнал Sheepfold;
- одинаковая бизнес-логика управления в LuCI и Android.

## Сборка

Быстрые тестовые IPK:

```powershell
python scripts/build-test-ipk.py --variant sheepfold
python scripts/build-test-ipk.py --variant sheepfoldAi
python scripts/build-test-ipk.py --variant all
```

Канонические release-IPK и OpenWrt APK:

```text
GitHub Actions → Build OpenWrt packages → Run workflow
```

Подробности: [`github-actions-openwrt-build.ru.md`](github-actions-openwrt-build.ru.md)
(§owrtci1).

Android:

```powershell
android\gradlew.bat -p android assembleDebug
android-child\gradlew.bat -p android-child assembleDebug
```

Обычная Gradle-сборка оставляет APK в `app/build/outputs/apk/debug/app-debug.apk`. Явные задачи `exportDebugApk` и `exportChildDebugApk` копируют их в заданный каталог только по прямому запросу; обычная проверка не пишет в Downloads.

## Проверки

- `tests/productVariants.test.mjs` распаковывает оба быстрых IPK и проверяет физическую границу AI-backend;
- `tests/openWrtVariantFeed.test.mjs` проверяет оба feed, которые получает официальный SDK;
- `tests/openWrtBuildWorkflow.test.mjs` защищает полную IPK/APK-матрицу, закрепление Action и release-permissions;
- тот же тест запрещает возвращать Android product flavors;
- `scripts/verify-android-variant.py` подтверждает присутствие server-gated AI-клиента в двух APK;
- CI проверяет два Android APK, а не четыре Android-варианта;
- тесты проверяют `capabilities.aiAssistant`, `childAiAvailable` и безопасное значение по умолчанию `false`.
- тест распаковывает оба IPK и подтверждает общее внутреннее имя пакета при разных именах файлов;
- тест проверяет резервную копию конфига и принудительную переустановку только при смене варианта на той же версии.

## Критерий готовности

- для роутера собраны четыре файла: две редакции в форматах IPK/OpenWrt APK;
- Android выпускает только два приложения: родительское и детское;
- обычная редакция роутерного пакета не обслуживает `/ai-assistant` и не содержит AI-модулей;
- AI Support содержит общий контроль и AI-добавки;
- родительский и детский Android APK едины для обеих редакций роутерного пакета;
- без положительного ответа роутера AI UI полностью скрыт;
- детский backend повторно проверяет согласие и личную группу;
- переход между редакциями сохраняет устройства, списки, группы и расписания;
- повторный переход сохраняет неактивные AI-настройки, но Standard не запускает AI-backend и сбор активности.
