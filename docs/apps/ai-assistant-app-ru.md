# Sheepfold AI Assistant (ИИ помощник)

## 1. Назначение

ИИ помощник Sheepfold — подключаемый модуль, который через API на роутере общается с внешней нейросетью и с родительскими/детскими устройствами. Он помогает:
- объяснять настройки и ограничения человеческим языком;
- по месту предлагать, как настроить правила и расписания;
- анализировать логи ребёнка и формировать осмысленные отчёты.

## 2. Целевая аудитория

- Родители и опекуны (чат с ИИ, советы по настройке).
- Дети (объяснения «почему заблокировано» в мягкой форме, если разрешено).
- Дизайнеры conversational‑UX и UI, куда встраивается ИИ.
- Тестировщики, проверяющие корректность контекста и ролей.

## 3. Основные функции

- Приём сообщений с устройств через роутерный API + передача роли отправителя (родитель/ребёнок), устройства и сетевого контекста.
- Объяснение настроек Sheepfold (правила, расписания, списки сайтов).
- Помощь в настройке: «поставь безопасные ограничения для игр», «разберись, почему ребёнок не может зайти».
- Анализ логов ребёнка (посещения, попытки доступа) с выдачей объясняющих и аккуратных ответов.

## 4. Для дизайнеров

### 4.1. Каналы

- Чат в родительском Android-приложении.
- Чат/экран объяснений в детском приложении.
- Возможная интеграция в мессенджер‑бота (как фронтэнд для ИИ).

### 4.2. Ролевой контекст

- Разные тон и уровень подробностей для родителя и ребёнка.
- В UI явно показывать, кто сейчас «говорит» с ИИ.
- Не допускать, чтобы ребёнок через ИИ инициировал действия, требующие прав родителя.

### 4.3. Карта диалогов

- Чат родителя: вопросы «как настроить…», «разберись в ситуации», «проанализируй логи».
- Чат ребёнка: объяснения «почему заблокировано», «когда снова будет доступ».
- Контекстные подсказки рядом с настройками (ассистент по месту).

## 5. Для тестировщиков

### 5.1. Ролевые сценарии

- Проверка, что при запросе от родителя в ИИ уходит role=parent + нужный контекст.
- Проверка, что при запросе от ребёнка ИИ не предлагает действия типа «сними блокировку».

### 5.2. Настройка и анализ

- Запрос «помоги настроить фильтр игр» → ответы ИИ согласованы с реальными возможностями Sheepfold.
- Запрос анализа логов → ИИ получает агрегированные данные, не раскрывает «лишнего», даёт понятное резюме.

### 5.3. Негативные кейсы

- Попытки «как обойти фильтр» → корректный отказ и объяснение.
- Ошибки связи с ИИ‑платформой → понятные сообщения пользователю и безопасный fallback.

## 6. Связь с другими приложениями

- Работает поверх уже существующих API/настроек и данных роутера.
- Получает контекст от родительских и детских приложений, а также, при необходимости, от бота.
