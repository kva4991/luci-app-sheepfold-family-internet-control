# Sheepfold AI Assistant (ИИ помощник)

## 1. Назначение

ИИ помощник Sheepfold — подключаемый модуль, который через API на роутере общается с внешней нейросетью и с родительскими/детскими устройствами.[cite:80][cite:110] Он помогает:
- объяснять настройки и ограничения человеческим языком;
- по месту предлагать, как настроить правила и расписания;
- анализировать логи ребёнка и формировать осмысленные отчёты.

## 2. Целевая аудитория

- Родители и опекуны (чат с ИИ, советы по настройке).
- Дети (объяснения «почему заблокировано» в мягкой форме, если разрешено).
- Дизайнеры conversational‑UX и UI, куда встраивается ИИ.[cite:110]
- Тестировщики, проверяющие корректность контекста и ролей.

## 3. Основные функции

- Приём сообщений с устройств через роутерный API + передача роли отправителя (родитель/ребёнок), устройства и сетевого контекста.[cite:110]
- Объяснение настроек Sheepfold (правила, расписания, списки сайтов).[cite:110]
- Помощь в настройке: «поставь безопасные ограничения для игр», «разберись, почему ребёнок не может зайти».[cite:110]
- Анализ логов ребёнка (посещения, попытки доступа) с выдачей объясняющих и аккуратных ответов.

## 4. Способы взаимодействия с ИИ

### 4.1. Через Android-приложение родителя

На стороне Android есть клиент `AiAssistantClient` и модель `AiAssistantRequest`, которые отправляют запросы к endpoint’у `/cgi-bin/sheepfold-api/ai-assistant` на роутере.[cite:110] При этом:

- используется URL `.../ai-assistant` на основе `apiUrl` из роутерного подключения;
- телефон **не ходит напрямую** к DeepSeek/Gemini и не хранит их API-ключи — всё идёт через Sheepfold API на роутере;[cite:110]
- в запросе передаётся контекст соединения (адрес роутера, роль, состояние интернета).

ИИ помощник может быть представлен:

- отдельной вкладкой/табом на главном экране Android (см. `android/README.md` — «parent AI assistant tab»);[cite:110]
- контекстными подсказками и диалогами в экранах настроек (расписания, списки сайтов).

### 4.2. Через Android-приложение ребёнка

Child‑приложение может отображать ответы ИИ в виде объяснений статуса и ограничений, если это включено в настройках:[cite:98][cite:110]

- запросы к ИИ проходят также через роутерный API, с явной пометкой роли `child`;
- ответы ИИ для ребёнка должны быть мягкими, понятными и не давать обходных путей.

### 4.3. Через LuCI и браузер

В LuCI‑интерфейсе есть строки и описания, связанные с «AI context for APK» и «AI assistant model/provider» (`overview.js`).[cite:110]

Там ИИ помощник появляется как:

- часть настроек контекста, который Android-приложение может запросить у роутера и показать родителю **до** отправки ИИ‑провайдеру (диагностический снимок без паролей, токенов, имён детей и MAC‑адресов);[cite:110]
- поле выбора модели и провайдера ИИ («AI assistant model», «AI provider», например, Gemini Free).

### 4.4. Через CGI endpoint на роутере

На роутере есть функция `ai_assistant_json()` в `sheepfold-api`, которая обрабатывает запросы ИИ помощника и возвращает JSON‑ответ.[cite:110]

Контракт описан в `docs/android-openwrt-api.ru.md`:

- текущий пакет уже включает ранний endpoint `/cgi-bin/sheepfold-api/ai-assistant` как заглушку для проверки Android‑соединения;[cite:110]
- полный authenticated API `/api/v1/*` и полноценное role‑based authorization ещё не завершены, поэтому этот endpoint рассматривается как экспериментальный.[cite:110]

### 4.5. Документация и промпты

Имеются отдельные документы: [cite:110]

- `docs/ai-assistant.md` и `docs/ai-assistant.ru.md` — продуктовый взгляд на ИИ помощника и передачу контекста;
- `docs/ai-context-sharing*.md` — правила, какие поля контекста могут передаваться ИИ и как подтверждать это в UI;
- черновик промпта ИИ‑помощника для поддержки родителя (`docs/ai-assistant-prompt-for-support-parent/v1/ai-assistant-prompt.ru.md`).

Эти файлы задают рамки того, какие данные можно отправлять ИИ‑провайдеру и как объяснять это пользователю.

## 5. Для дизайнеров

### 5.1. Каналы

- Чат в родительском Android-приложении (AiAssistant tab).[cite:110]
- Чат/экран объяснений в детском приложении.
- Интеграция ИИ‑подсказок в LuCI (например, пояснения к диагностическому снимку и настройкам).[cite:110]

### 5.2. Ролевой контекст

- Разные тон и уровень подробностей для родителя и ребёнка.
- В UI явно показывать, кто сейчас «говорит» с ИИ.
- Не допускать, чтобы ребёнок через ИИ инициировал действия, требующие прав родителя.

### 5.3. Карта диалогов

- Чат родителя: вопросы «как настроить…», «разберись в ситуации», «проанализируй логи».
- Чат ребёнка: объяснения «почему заблокировано», «когда снова будет доступ».
- Контекстные подсказки рядом с настройками (ассистент по месту).

## 6. Для тестировщиков

### 6.1. Ролевые сценарии

- Проверка, что при запросе от родителя в ИИ уходит role=parent + нужный контекст (через `AiAssistantRequest` и `/cgi-bin/sheepfold-api/ai-assistant`).[cite:110]
- Проверка, что при запросе от ребёнка ИИ не предлагает действия типа «сними блокировку» и не раскрывает лишние детали.[cite:110]

### 6.2. Настройка и анализ

- Запрос «помоги настроить фильтр игр» → ответы ИИ согласованы с реальными возможностями Sheepfold и не нарушают правила безопасности (`AI_NOTICE.md`, `docs/development-ideas*.md`).[cite:109][cite:110]
- Запрос анализа логов → ИИ получает агрегированные данные, не раскрывает токены/пароли/личные данные и даёт понятное резюме.[cite:109]

### 6.3. Негативные кейсы

- Попытки «как обойти фильтр» → корректный отказ и объяснение.[cite:110]
- Ошибки связи с ИИ‑платформой (таймаут, 5xx) → понятные сообщения пользователю и безопасный fallback (502, сообщения о `gemini_request_failed` в `sheepfold-api`).[cite:110]

## 7. Связь с другими приложениями

- Работает поверх уже существующих API/настроек и данных роутера (`sheepfold-api`, `router-info`, pairing).[cite:110]
- Взаимодействует с родительскими и детскими приложениями через `AiAssistantClient` и контекст-sharing.[cite:110]
- Может использовать данные из LuCI‑диагностики и других модулей для формирования более точного ответа.
