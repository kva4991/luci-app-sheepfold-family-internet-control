# Sheepfold Android Child (приложение ребёнка)

## 1. Назначение

Android-приложение для ребёнка (каталог `android-child`) — лёгкий клиент, который устанавливается на устройство ребёнка, запрашивает у роутера собственный статус доступа и показывает понятные ограничения, уведомления и разрешённый родителем ИИ-помощник.

Приложение не передаёт на роутер MAC-адрес, имя устройства или собственный “статус”. Роутер сам определяет клиента по `REMOTE_ADDR -> MAC -> UCI device` через endpoint `/cgi-bin/sheepfold-api/client-status` (§b5wkq2e).

## 2. Целевая аудитория

- Дети, использующие устройство под контролем родителей.
- Дизайнеры детского мобильного интерфейса.
- Тестировщики, проверяющие корректность взаимодействия child-приложения с роутером.

## 3. Основные функции

- Получение собственного клиентского статуса от роутера через `sheepfold-api-client-status` (§b5wkq2e).
- Отображение локальных уведомлений о блокировке/ограничении.
- Возможное отображение таймера/счётчика до конца разрешённого времени.

## 4. Для дизайнеров

### 4.1. Принципы дизайна

- Интерфейс должен быть простым и понятным для ребёнка.
- Минимум управляющих элементов: фокус на отображении статуса и сообщений.
- Сдержанная визуализация ограничений (без излишней тревожности), но с ясностью, что доступ ограничен.

### 4.2. Экраны

- Экран статуса: онлайн/офлайн, доступ разрешён/ограничен.
- Экран сообщения от родителя (если реализовано).
- Экран таймера до конца сессии.

### 4.3. Карта экранов и функций

1. **Экран статуса (Status)**
   - Индикация состояния соединения с роутером (есть связь / нет связи).
   - Индикация состояния доступа: «Доступ разрешён», «Доступ ограничен», «Полная блокировка».

2. **Экран сообщений (Messages)**
   - Отображение текстового сообщения от родителя/админа (например, причина блокировки, инструкции).
   - Возможные визуальные элементы (иконки, цвета), подчёркивающие важность сообщения.

3. **Экран таймера (Timer)**
   - Отображение оставшегося времени до конца разрешённой сессии.
   - Простая анимация/обновление таймера без перегрузки ребёнка.

## 5. Для тестировщиков

- Проверка, что приложение не отправляет IP/MAC/статус как доверенные данные, а роутер сам определяет устройство по текущему сетевому подключению (§b5wkq2e).
- Проверка отображения сообщений при разных сценариях (полная блокировка, временный доступ, списки сайтов).
- Проверка работы при отсутствии связи с роутером.

### 5.1. Карта тестовых сценариев по экранам

1. **Status**
   - Правильное отображение состояния доступа при смене политики на роутере (разрешение, ограничение, блокировка).
   - Проверка поведения UI при потере связи с роутером.

2. **Messages**
   - Отображение различных типов сообщений (информационные, предупреждения, инструкции).
   - Проверка локализации/языка сообщений, если предусмотрено.

3. **Timer**
   - Проверка корректного уменьшения таймера и реакции на его завершение.
   - Проверка поведения при изменении оставшегося времени со стороны роутера.

## 6. Связь с другими приложениями

- Child-приложение не управляет правилами и не является источником доверенных данных о себе. Управление идёт через роутер, LuCI, родительское приложение и разрешённый мессенджер, а child-приложение только показывает статус, который вернул роутер (§b5wkq2e).
