# Sheepfold боты (Telegram / VK)

## 1. Назначение

Боты Sheepfold (каталог `bot`, а также скрипт `sheepfold-telegram-bot`) позволяют управлять семейным контролем через мессенджеры (Telegram, при наличии — VK). Они дают родителям возможность выполнять быстрые операции без входа в LuCI или мобильное приложение.[cite:57]

## 2. Целевая аудитория

- Родители и администраторы, предпочитающие управлять через мессенджер.
- Дизайнеры conversational-UX (диалоги бота).
- Тестировщики, проверяющие сценарии команд и ответы бота.

## 3. Основные функции

- Получение текущего статуса сети и устройств.
- Блокировка/разблокировка устройств или всего интернета.
- Настройка временного доступа.
- Получение уведомлений о новых устройствах и событиях (например, попытках доступа).[cite:57]

## 4. Для дизайнеров

### 4.1. Диалоговые сценарии

- Команды должны быть понятны и коротки (например, `/status`, `/block <device>`, `/allow <device>`).
- Ответы бота — структурированные: краткий заголовок + перечень устройств/состояний.
- Важные события (глобальная блокировка, новые устройства) выделять визуально (эмодзи, форматирование), но без перегрузки.

### 4.2. Контекст

- Бот должен объяснять последствия команд (например, «глобальная блокировка выключит интернет для всех устройств»).[cite:58]
- При ошибках (неизвестное устройство, недоступен роутер) сообщения должны быть дружественными и информативными.

### 4.3. Карта диалогов и команд

1. **Статус сети и устройств**
   - Команда `/status` или аналогичная: краткий отчёт о состоянии интернета и количестве устройств.[cite:57]
   - Детализированный список устройств по группам (дети, родители, "Без ограничений") при необходимости.[cite:61]

2. **Блокировка/разблокировка устройств**
   - Команды `/block <device>` и `/allow <device>` (или меню-кнопки), которые меняют политику конкретного устройства.[cite:58]
   - Ответы с подтверждением действия и указанием последствий (например, «Устройство X заблокировано, интернет недоступен»).

3. **Глобальная блокировка интернета**
   - Команда для включения/выключения глобальной блокировки (например, `/block_all`, `/unblock_all`).[cite:58]
   - Ответы с явным предупреждением о влиянии на всю сеть.

4. **Временный доступ**
   - Команда для выдачи временного доступа устройству (например, `/temp_allow <device> <time>`).
   - Ответы, показывающие срок временного доступа и способ его отмены.

5. **Уведомления и алерты**
   - Автоматические сообщения о новых устройствах, попытках доступа, изменении WAN-состояния.[cite:57]
   - Возможно, отдельные команды для просмотра журнала событий (например, `/events`).

## 5. Для тестировщиков

- Проверка обработки корректных и некорректных команд.
- Проверка реакций на отсутствие связи с роутером.
- Проверка сценариев безопасности (требование токена/аутентификации для привилегированных команд).[cite:57]

### 5.1. Карта тестовых сценариев по диалогам

1. **Статус**
   - Проверка того, что `/status` корректно отражает текущее состояние сети и устройств.[cite:58]
   - Проверка поведения при недоступности роутера (сообщение об ошибке, отсутствие устаревших данных).

2. **Блокировка/разблокировка**
   - Проверка, что команды `/block`/`/allow` меняют политику устройства и это отражается в LuCI/мобильных приложениях.[cite:58]
   - Проверка сообщений при попытке блокировки неизвестного устройства.

3. **Глобальная блокировка**
   - Проверка включения/выключения глобальной блокировки и соответствующих уведомлений.[cite:58]
   - Проверка реакций на повторные команды или конфликтные запросы.

4. **Временный доступ**
   - Проверка выдачи временного доступа через бота и его окончания по времени.
   - Проверка отмены временного доступа и соответствующих сообщений.

5. **Уведомления/алерты**
   - Проверка автоматических сообщений при ключевых событиях (новые устройства, смена WAN-состояния).[cite:57]
   - Проверка того, что уведомления не содержат чувствительных данных (например, токенов).

## 6. Связь с другими приложениями

- Боты используют команды контроллера (`sheepfold-router-control`) и API (`sheepfold-api-*`).[cite:58][cite:60]
- Изменения, внесённые через бота, должны быть видны в LuCI и мобильных приложениях.
