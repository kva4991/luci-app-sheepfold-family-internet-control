# Sheepfold Android (приложение родителя)

## 1. Назначение

Android-приложение (каталог `android`) — клиент для родителей/опекунов, который подключается к Sheepfold на роутере и позволяет управлять устройствами, правилами и отчётами с мобильного телефона.[cite:80]

## 2. Целевая аудитория

- Родители, контролирующие интернет-доступ детей.
- Дизайнеры мобильных интерфейсов (Material/Compose), работающие над UX панелей управления.
- Тестировщики мобильного приложения.

## 3. Основные функции

- Подключение к роутеру через pairing-код и Bearer-токен.[cite:58]
- Просмотр списка устройств и их статуса (онлайн/офлайн, группа, политика).
- Управление allowlist/blocklist и временным доступом.
- Настройка расписаний и списков сайтов.
- Получение уведомлений о новых устройствах и событиях.

## 4. Для дизайнеров

### 4.1. Навигация и экраны

- Экран подключения/поддержки нескольких роутеров.
- Главный экран: сводка по сети, критичные события, быстрые действия.
- Экран устройств: карточки устройств с типом, группой и статусом.
- Экран правил: расписания, списки сайтов, глобальные настройки.
- Экран журналов/отчётов.

Важно:

- Чётко показывать статус безопасности (например, глобальная блокировка включена, устройство заблокировано).
- Учитывать роли пользователей (основной родитель vs другие администраторы).
- Предусматривать понятные подтверждения для destructive-действий (блокировка, формат USB). [cite:57]

### 4.2. Визуальные элементы

- Иконки типов устройств (телефон, планшет, ноутбук, ТВ).
- Цветовые маркеры групп (детская, родительская, "Без ограничений").[cite:61]
- Состояния кнопок для временного доступа (активно, истекло, готово к продлению).

### 4.3. Карта экранов и функций

1. **Экран подключения (Onboarding / Pairing)**
   - Ввод адреса роутера или поиск в локальной сети.
   - Ввод pairing-кода с роутера.[cite:58]
   - Отображение прогресса подключения и ошибок (неверный код, нет связи).[cite:60]

2. **Главный экран (Dashboard)**
   - Карточка состояния сети: интернет онлайн/limited/offline, глобальная блокировка.[cite:58]
   - Виджет «Новые устройства» с быстрыми действиями (добавить в группу, заблокировать).
   - Быстрые кнопки: включить/выключить глобальную блокировку, открыть список устройств, перейти к правилам.

3. **Экран устройств (Devices)**
   - Список устройств с фильтрами по группам, статусу онлайн/офлайн и блокировке.[cite:61]
   - Детальная карточка устройства: информация, группа, текущая политика, историю событий.
   - Действия: перевести в другую группу, добавить/убрать из allowlist/blocklist, включить временный доступ.

4. **Экран правил (Policies)**
   - Управление глобальной политикой (например, разрешены/запрещены определённые категории сайтов).[cite:57]
   - Настройка расписаний доступа для групп и устройств.
   - Управление списками сайтов (allowlist/blocklist) с поиском и импортом.

5. **Экран журналов/отчётов (Logs / Reports)**
   - История событий: подключение устройств, блокировки, изменение правил.
   - Фильтры по типу событий и устройствам.
   - Возможный экспорт/шаринг отчётов.

## 5. Для тестировщиков

### 5.1. Сценарии pairing и токенов

- Правильная работа pairing-процесса с роутером: ввод кода, получение токена, сохранение в защищённом хранилище.[cite:58]
- Обработка неправильного, истёкшего или повторно используемого pairing-кода.
- Проверка того, что токен не хранится в открытом виде и не попадает в журналы.

### 5.2. Сетевые сценарии

- Потеря соединения с роутером, переход в оффлайн-режим.
- Обработка ошибок API (401, 403, 500) на уровне UI.

### 5.3. Карта тестовых сценариев по экранам

1. **Onboarding / Pairing**
   - Успешное подключение к роутеру с корректным pairing-кодом.[cite:58]
   - Ошибки: неверный код, истёкший код, роутер недоступен.
   - Повторное использование pairing-кода — ожидаемое поведение (отказ/повторное привязывание).

2. **Dashboard**
   - Отображение правильного WAN-состояния и глобальной блокировки при смене состояния на роутере.[cite:58]
   - Корректное отображение новых устройств и быстрых действий по ним.

3. **Devices**
   - Проверка, что изменения групп/политик в UI отражаются в поведении роутера (allowlist/blocklist, временный доступ).[cite:61]
   - Проверка фильтров и поиска по устройствам.

4. **Policies**
   - Настройка расписаний и проверка смены политики по времени.
   - Настройка списков сайтов и проверка блокировки/разрешения доменов.

5. **Logs / Reports**
   - Проверка записи ключевых событий и отображения их в интерфейсе.
   - Проверка реакции UI на недоступность журнала/ошибки API.

## 6. Связь с другими приложениями

- Использует роутерный API (`sheepfold-api-*`) для всех операций.[cite:60]
- Может отображать события, пришедшие через Telegram/VK-боты, если такая интеграция предусмотрена.
