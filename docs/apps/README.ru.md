# Документация приложений Sheepfold (docs/apps)

В каталоге `docs/apps` хранятся русскоязычные описания всех основных приложений, входящих в экосистему Sheepfold:

- серверное приложение на роутере (LuCI / OpenWrt);
- Android-приложение родителя;
- Android-приложение ребёнка;
- боты (Telegram / VK).

## Для кого эти документы

Документация в `docs/apps` предназначена для:

- **дизайнеров** — чтобы понимать структуру экранов, пользовательские сценарии и визуальные паттерны каждого приложения;
- **тестировщиков** — чтобы иметь карту функциональных и негативных сценариев по экранам/диалогам;
- **продуктовых и технических авторов** — чтобы не потерять архитектурное и UX-видение проекта при дальнейшей разработке.

## Зачем они нужны

Эти файлы дополняют основные README/AGENTS/CODING_RULES и фиксируют предметную область с точки зрения конкретных приложений:

- описывают роли пользователей и основные потоки работы;
- дают «книжное» оглавление по темам (назначение, аудитория, функции, для дизайнеров, для тестировщиков, карты экранов/сценариев);
- помогают проектировать новые функции и интерфейсы без потери контекста;
- служат базой для формирования более формальных PRD/ТЗ и тест-планов.

## Как обновлять

При добавлении новых экранов, функций или интеграций:

- расширять соответствующий файл в `docs/apps` новыми разделами и картами экранов/сценариев;
- следить, чтобы описания оставались согласованными с фактическим поведением приложений и API;
- при крупных изменениях продукта добавлять ссылки на эти файлы в основной `README.ru.md` или другую верхнеуровневую документацию.
