# Безопасное обновление Sheepfold

<!-- §updsafe -->

## Граница гарантии

Updater устанавливает только более новую stable-версию выбранного варианта Sheepfold и старается не оставлять настройки в частично изменённом состоянии. Он **не выполняет бинарный rollback**: для такого отката нужен реально сохранённый и проверенный предыдущий IPK, которого текущая схема ещё не гарантирует.

## Что проверяется до `opkg`

1. Ответ читается из GitHub `releases/latest`.
2. `browser_download_url` обязан вести в `https://github.com/kva4991/luci-app-sheepfold-family-internet-control/releases/download/` и содержать имя именно установленного варианта: Standard либо AI Support.
3. Сетевая попытка ограничена 120 секундами, файл — 64 МиБ. Реальный код `uclient-fetch`, `curl` или `wget` сохраняется.
4. Внешний gzip-tar обязан содержать ровно по одному `debian-binary`, `control.tar.gz` и `data.tar.gz` без небезопасных путей и повторяющихся файлов.
5. `debian-binary` обязан содержать `2.0`.
6. В `control` допускается ровно по одному полю `Package`, `Version`, `Architecture`. Имя совпадает с выбранным продуктом, версия — с именем release asset, архитектура — `all`.
7. Оба вложенных архива должны полностью читаться; абсолютные пути и компоненты `..` запрещены. Повторные записи каталогов разрешены, повторный файл запрещён.

Только после этих проверок запускается `opkg install`.

## Поведение при установке

- Перед `opkg` updater запоминает, существовал ли `/etc/config/sheepfold`, и создаёт копию с правами `0600`.
- `preinst` дополнительно сохраняет последний непустой конфиг в `/etc/sheepfold/migrations/sheepfold.config.pre-upgrade`.
- `postinst` не удаляет оперативную копию: это делает updater после того, как `opkg` вернул результат.
- При ненулевом коде `opkg` updater возвращает тот же код и восстанавливает прежний конфиг. Если конфиг до обновления отсутствовал, появившийся после неудачной установки файл удаляется.
- Даже после кода `0` updater повторно читает установленную версию через `opkg status`. Несовпадение считается ошибкой и вызывает восстановление конфига.
- После подтверждённой версии временный IPK и копия удаляются, затем перезапускаются Sheepfold и `rpcd`.

## Почему недостаточно `set -e`

Shell-функция, вызванная в конструкции `install_update || code=$?`, может выполняться в контексте, где `set -e` не останавливает её на каждой внутренней ошибке. Поэтому валидация, backup, `opkg` и проверка версии имеют явные ветки возврата. Этот контракт защищён поведенческим тестом `tests/updateTransportSafety.test.mjs`.

## Что ещё требуется

- Проверка на живом актуальном OpenWrt с настоящим `uclient-fetch`, BusyBox `tar`, `opkg` и `ash`.
- Проверка нехватки `/tmp`, обрыва питания и ошибки реального `postinst`.
- Решение о хранении предыдущего IPK и безопасном бинарном rollback. Пока архив не сохранён заранее, не повреждён и не соответствует предыдущей установленной версии, кнопку `Откатить` показывать нельзя.
