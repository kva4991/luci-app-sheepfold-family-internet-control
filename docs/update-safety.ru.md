# Безопасное обновление Sheepfold

<!-- §updsafe -->

## Граница гарантии

Updater устанавливает только более новую stable-версию выбранного варианта Sheepfold и старается не оставлять настройки в частично изменённом состоянии. Он поддерживает IPK/`opkg` на OpenWrt 24.10 и старше и OpenWrt APK/`apk` v3 на 25.12 и новее через единый адаптер (§pkgmgr1). Он **не выполняет бинарный rollback**: для такого отката нужен реально сохранённый и проверенный предыдущий пакет, которого текущая схема ещё не гарантирует.

## Что проверяется до менеджера пакетов

1. Ответ читается из GitHub `releases/latest`.
2. `browser_download_url` обязан вести в `https://github.com/kva4991/luci-app-sheepfold-family-internet-control/releases/download/` и содержать имя именно установленного варианта: Standard либо AI Support.
3. Сетевая попытка ограничена 120 секундами, файл — 64 МиБ. Реальный код `uclient-fetch`, `curl` или `wget` сохраняется.
4. Для IPK внешний gzip-tar обязан содержать ровно по одному `debian-binary`, `control.tar.gz` и `data.tar.gz` без небезопасных путей и повторяющихся файлов; `debian-binary` содержит `2.0`.
5. Для IPK в `control` допускается ровно по одному полю `Package`, `Version`, `Architecture`. Имя совпадает с устанавливаемой сущностью, версия — с release asset, архитектура — `all`.
6. Оба вложенных IPK-архива должны полностью читаться; абсолютные пути и компоненты `..` запрещены. Повторные записи каталогов разрешены, повторный файл запрещён.
7. Для OpenWrt APK выполняются `apk verify --allow-untrusted` и `apk adbdump --format json`. Из внутренних метаданных через `jshn` проверяются точные `info.name`, `info.version` и совместимая `info.arch`; проверка не полагается только на имя файла.
8. Строгий префикс OpenWrt-пакета не позволяет принять родительский или детский Android APK за пакет роутера.

Только после этих проверок единый адаптер вызывает `opkg install` или `apk add --allow-untrusted`.

## Поведение при установке

- Перед установкой updater запоминает, существовал ли `/etc/config/sheepfold`, и создаёт копию с правами `0600`.
- `preinst` дополнительно сохраняет последний непустой конфиг в `/etc/sheepfold/migrations/sheepfold.config.pre-upgrade`.
- `postinst` не удаляет оперативную копию: это делает updater после того, как системный менеджер пакетов вернул результат.
- При ненулевом коде установки updater возвращает тот же код и восстанавливает прежний конфиг. Если конфиг до обновления отсутствовал, появившийся после неудачной установки файл удаляется.
- Даже после кода `0` updater повторно читает установленную версию через общий адаптер. Несовпадение считается ошибкой и вызывает восстановление конфига.
- После подтверждённой версии временный пакет и копия удаляются, затем перезапускаются Sheepfold и `rpcd`.

## Почему недостаточно `set -e`

Shell-функция, вызванная в конструкции `install_update || code=$?`, может выполняться в контексте, где `set -e` не останавливает её на каждой внутренней ошибке. Поэтому валидация, backup, установка и проверка версии имеют явные ветки возврата. Этот контракт защищён поведенческим тестом `tests/updateTransportSafety.test.mjs`; в нём отдельно проверяются IPK/`opkg` и APK/`apk`.

## Что ещё требуется

- Проверка на живых OpenWrt 24.10 и 25.12 с настоящими `uclient-fetch`, BusyBox `ash`, `opkg` и `apk` v3.
- Проверка нехватки `/tmp`, обрыва питания и ошибки реального `postinst`.
- Решение о хранении предыдущего пакета и безопасном бинарном rollback. Пока архив не сохранён заранее, не повреждён и не соответствует предыдущей установленной версии, кнопку `Откатить` показывать нельзя.
