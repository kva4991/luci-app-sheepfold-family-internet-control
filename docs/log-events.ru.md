# Журнал Sheepfold

Команды, выполненные из LuCI, получают явный источник `LuCI (Изменено на роутере)`.
Android, Telegram, фоновые службы и WPS передают собственный источник и не должны
ошибочно записываться как LuCI (§logaudit).

Журнал хранится в RAM по пути из настройки `log_cache_path`, по умолчанию `/tmp/sheepfold/events.log`.
Формат строки: `дд.мм.гггг чч:мм:сс<TAB>сообщение`.

После перезагрузки роутера RAM-журнал **очищается**. Если в `sheepfold.global.log_storage` выбраны USB или Yandex Disk, новые события **дополнительно зеркалируются** на внешнее хранилище (`sheepfold-log` → `mirror_to_usb` / `schedule_yandex_push`). Подробности: [`docs/usb-storage-design.ru.md`](usb-storage-design.ru.md), [`docs/yandex-disk-storage.ru.md`](yandex-disk-storage.ru.md).

Этот журнал предназначен для административных и технических событий. Он не должен превращаться в историю посещённых сайтов.

Если история интернет-активности устройств будет добавлена позже, она должна быть отдельной функцией с отдельной настройкой, отдельной страницей, отключением по умолчанию и исключением устройств администраторов и белого списка. Подробности: `docs/site-activity-logs.ru.md`.

## Сейчас пишутся

- `Обнаружено новое устройство: <MAC>, IP <IP>, имя <имя>` — при первом появлении MAC в данных роутера.
- `Устройство добавлено вручную: <MAC>, IP <IP>` — когда администратор вручную добавил известное устройство по MAC.
- `Устройство добавлено в белый список: <MAC>, IP <IP>` — когда администратор добавил устройство в белый список через LuCI/backend.
- `Устройство добавлено в чёрный список: <MAC>, IP <IP>` — когда администратор добавил устройство в чёрный список через LuCI/backend.
- `Устройство <имя/MAC>, IP <IP> автоматически добавлено в группу "Без ограничений": <причина>` — когда автоопределение доверенной инфраструктуры сработало достаточно уверенно.
- `Устройство <MAC>, IP <IP> пыталось открыть настройки роутера, но доступ был заблокирован` — когда неадминское и не белое устройство попало на blocked-page responder при попытке открыть LuCI/админский URL роутера.
- `У роутера пропал интернет: кабель WAN отключён или порт без линка` — когда WAN-линк физически пропал.
- `У роутера пропал интернет: WAN-интерфейс недоступен` — когда OpenWrt считает WAN недоступным.
- `У роутера пропал интернет: WAN-интерфейс не поднят` — когда WAN не находится в состоянии `up`.
- `У роутера пропал интернет: таймаут проверки связи` — когда WAN поднят, но контрольные IP не отвечают.
- `У роутера пропал интернет: не удалось получить статус WAN через ubus` — когда backend не смог получить статус WAN.
- `Интернет на роутере восстановлен: проверка связи прошла успешно` — когда WAN снова работает.
- `Wi-Fi включён Sheepfold` — когда Sheepfold включает все Wi-Fi радиомодули.
- `Wi-Fi отключён Sheepfold` — когда Sheepfold отключает все Wi-Fi радиомодули.
- `WPS-подключение разрешено на 30 секунд` — когда WPS PBC успешно запущен.
- `Не удалось запустить WPS-подключение на роутере` — когда WPS PBC не удалось запустить.
- `Окно WPS-добавления в белый список завершено` — после завершения 30-секундного окна WPS allowlist.
- `Устройство #<ID> <имя> использует SIM-карту: SIM <слот>: <номер или "номер недоступен"> (обнаружена в телефоне при установке приложения)` — при первом отчёте детского APK, если SIM уже установлена (§simchg1).
- `Устройство #<ID> <имя> сменило SIM-карту: SIM <слот>: <номер или "номер недоступен">` — при последующем изменении набора активных SIM; только первый пустой отчёт создаёт исходную точку без события (§simchg1).
- `Устройство #<ID> <имя> подключилось к новой Wi-Fi сети «<SSID>».` — при первом отчёте о новом отпечатке сети; при отдельно разрешённой геолокации добавляется последнее доступное местоположение телефона и точность (§childwifi1).
- `Администратор очистил историю новых Wi-Fi сетей детских устройств` — после удаления сохранённой истории из вкладки `Уведомления` (§childwifi1).

## Полезно добавить дальше

- кто из администраторов включил/выключил интернет;
- кто удалил устройство из белого или чёрного списка;
- кто изменил расписание, группу, аварийно-полезные сайты, WPS, Wi-Fi или LED-настройки;
- кто импортировал/экспортировал настройки;
- результат обновления приложения;
- попытки использовать просроченный или уже использованный QR/token сопряжения;
- частые ошибки AdGuard Home/Podkop-интеграции;
- нехватка места в `/tmp` или невозможность записать журнал.
