# Политика приватности и обработки данных

Дата редакции: 2026-07-02

Этот документ описывает, какие данные может обрабатывать Sheepfold, где они хранятся и когда могут передаваться наружу.

Sheepfold проектируется как self-hosted инструмент для семьи: без обязательного облака разработчика и без скрытой телеметрии.

## Основной принцип

По умолчанию данные должны оставаться внутри домашней инфраструктуры:

- на OpenWRT-роутере;
- в Android-приложении родителя-администратора;
- в локальных резервных копиях, которые создаёт пользователь.

Данные не должны отправляться разработчику Sheepfold без явного действия пользователя.

## Где какие данные хранятся

### OpenWRT-роутер

На роутере могут храниться:

- MAC-адреса устройств;
- текущие и постоянные IP-адреса;
- hostname, сетевые имена и пользовательские названия устройств;
- принадлежность устройства к белому списку, чёрному списку или расписанию;
- правила временного доступа;
- настройки аварийно-полезных сайтов;
- страна профиля роутера;
- настройки Telegram, VK или экспериментального MAX, если включён мессенджер;
- технический журнал, если журналирование включено.

### Android-приложение родителя

Android-приложение предназначено только для родителей-администраторов и не должно устанавливаться на телефоны детей.

В приложении могут храниться:

- адрес роутера или способ связи с роутером;
- токен или сессия доступа к Sheepfold API;
- выбранный язык;
- локальные настройки приложения;
- API-ключ ИИ-провайдера, если пользователь включает ИИ-помощника;
- локальная история экрана или чата ИИ, только если такая функция явно включена.

Пароль root от роутера не должен храниться в Android-приложении.

### Telegram, VK или экспериментальный MAX

Если пользователь включает Telegram, VK или экспериментальный MAX, через выбранный мессенджер могут проходить:

- уведомления о событиях;
- команды администратора;
- ответы бота;
- названия устройств;
- IP-адреса или MAC-адреса, если пользователь разрешил показывать их в боте;
- идентификаторы чатов или пользователей, которым разрешено управление.

На одном роутере может быть активен только один мессенджер: Telegram, VK или экспериментальный MAX.

### ИИ-провайдер

ИИ-помощник не должен получать MAC-адреса, IP-адреса, имена детей, точные названия устройств, семейные детали, журнал действий или техническую конфигурацию роутера без отдельного явного согласия пользователя.

По умолчанию приложение должно отправлять ИИ только текст вопроса, который пользователь сам написал в чате помощника.

Перед отправкой чувствительного контекста приложение должно показать отдельное подтверждение.

## Логи

Журналирование должно быть настраиваемым.

Варианты срока хранения:

```text
6m
3m
1m
14d
7d
3d
1d
12h
3h
1h
off
```

Рекомендуемый дефолт для OpenWRT:

```text
3d
```

Причина: для обычной семьи с несколькими активными устройствами трёх дней обычно достаточно для диагностики, но такой срок меньше рискует замусорить память роутера.

Кроме срока хранения, должен быть лимит размера журнала. Рекомендуемый дефолт:

```text
1024 KB
```

Удаление старых записей должно происходить по первому достигнутому пределу: срок хранения или размер.

## Что писать в журнал

Писать только события, полезные для администрирования:

- вход администратора;
- изменение правил устройства;
- добавление или удаление устройства из белого/чёрного списка;
- выдача временного доступа;
- срабатывание расписания;
- импорт/экспорт настроек;
- очистка журнала;
- обновление приложения;
- перезагрузка роутера через Sheepfold;
- ошибки firewall, rpcd, UCI, AdGuard Home, Podkop, Telegram/VK/MAX;
- факт отправки уведомления, без токена и лишнего содержимого.

## Что маскировать

Рекомендуемый режим по умолчанию: `balanced`.

В режиме `balanced`:

- MAC-адрес показывать частично, например `AA:BB:CC:xx:xx:xx`;
- IP-адрес можно показывать полностью в локальном интерфейсе, но в экспортируемых логах маскировать последний октет, например `192.168.1.x`;
- chat ID / user ID мессенджера маскировать частично;
- токены, API-ключи, пароли и session ID всегда заменять на `[secret]`;
- названия устройств показывать в локальном интерфейсе, но давать опцию маскировать их при экспорте.

Экспорт логов должен иметь опцию:

```text
Экспортировать с маскировкой чувствительных данных
```

Эта опция должна быть включена по умолчанию.

## Что не писать вообще

Не писать в журнал:

- пароли;
- API-ключи;
- токены Telegram/VK/MAX;
- session cookies;
- полный текст переписки в Telegram/VK/MAX;
- полный текст запросов к ИИ-помощнику;
- полную историю посещённых сайтов;
- DNS-запросы всех устройств в стиле AdGuard Home query log;
- содержимое HTTP-запросов;
- банковские данные;
- медицинские данные;
- точные персональные сведения о детях;
- приватные сообщения и переписки.

Sheepfold не должен превращаться в систему скрытого мониторинга поведения ребёнка.

## Очистка и экспорт

В LuCI и Android должна быть кнопка:

```text
Очистить журнал
```

Перед очисткой нужен вопрос подтверждения.

Экспорт логов должен быть отдельной командой. По умолчанию экспорт должен маскировать чувствительные данные.

## Боты и администраторы

Управление через Telegram, VK или экспериментальный MAX разрешается только пользователям, явно добавленным администратором в настройках роутера.

У каждого администратора должен быть:

- локальный аккаунт или запись в Sheepfold;
- роль, если в будущей версии будет включено разделение прав;
- связанный Telegram/VK/MAX user ID или chat ID;
- журнал административных действий.

Журнал действий относится к действиям администраторов: кто выдал доступ, кто заблокировал устройство, кто изменил список, кто очистил журнал, кто запустил обновление.

## Настройки Wi-Fi

Sheepfold может показывать и менять общие настройки Wi-Fi 2.4 GHz и 5 GHz:

- название сети;
- пароль;
- режим защиты;
- канал.

Изменение Wi-Fi может отключить пользователей от сети, поэтому такие действия должны требовать подтверждения администратора.

## Клиентская заглушка

Для заблокированных устройств можно показывать простую страницу-заглушку вместо бесконечной загрузки сайта.

Заглушка должна быть информативной, но не должна раскрывать лишние административные данные.

Пример:

```text
Интернет временно недоступен по семейным правилам.
Если это ошибка, обратитесь к родителю.
```

Отдельный детский интерфейс не планируется.

## Магазины приложений

Публикация Android-приложения в магазинах приложений не является ближайшим этапом, но должна учитываться в архитектуре.

Перед публикацией нужно будет подготовить:

- публичную политику приватности;
- описание собираемых данных для карточки приложения;
- проверку соответствия правилам магазина приложений;
- понятное объяснение, что приложение предназначено для родителей-администраторов, а не для скрытой установки на телефоны детей.
