# Первичная настройка времени и необязательный `nmap`

<!-- §country1 §devpas1 §pkgmgr1 -->

Этот документ фиксирует исполняемые границы двух P1-сценариев: безопасной настройки часов роутера и честной деградации полного распознавания устройств без тяжёлого сканера портов.

## Время, часовой пояс и NTP

Sheepfold сначала читает настоящий `system.@system[0].zonename`, `system.@system[0].timezone` и `system.ntp`. Уже настроенные OpenWrt-значения имеют приоритет над заводскими рекомендациями Sheepfold.

Профиль страны даёт только предлагаемый набор:

| Профиль | Предложенный часовой пояс | Предложенные NTP-серверы |
|---|---|---|
| Россия | `Europe/Moscow`, `MSK-3` | `ntp1.vniiftri.ru`, `ntp2.ntp-servers.net`, `3.openwrt.pool.ntp.org` |
| Беларусь | `Europe/Minsk`, `+03-3` | `by.pool.ntp.org`, `pool.ntp.org` |
| Китай | `Asia/Shanghai`, `CST-8` | `cn.pool.ntp.org`, `ntp.aliyun.com`, `time1.cloud.tencent.com` |

Одна страна может иметь несколько часовых поясов. Поэтому:

- внешний IP не отправляется в сервис геолокации;
- профиль России не заменяет существующий регион на Москву;
- рекомендация применяется только после отдельного нажатия;
- запись выполняется только общей явной кнопкой сохранения;
- неизвестный или повреждённый timezone отклоняется, а не заменяется молча;
- список NTP должен состоять только из допустимых hostnames, частично повреждённый список не урезается незаметно.

`sheepfold-time-control save` использует общий kernel lock, изолированный UCI savedir, снимки `/etc/config/system` и `/etc/config/sheepfold`, один проверяемый переход и восстановление обоих снимков при ошибке. После успешной записи перезапускается только `sysntpd`, если он доступен.

`sheepfold-time-control status` сообщает, настроен ли system timezone и выглядит ли текущий год правдоподобно. Неверные часы показываются как предупреждение; этот статус сам не меняет правила доступа.

## Полный режим без проверки портов

Основное определение устройств продолжает использовать DHCP, neighbor/ARP, hostapd, mDNS, SSDP/UPnP, WS-Discovery, OUI и сохранённые родителем признаки. `nmap` остаётся только дополнительным ограниченным источником слабых подсказок для серверов, камер, принтеров и похожих устройств.

Когда выбран `Полный режим`, но `nmap` отсутствует, LuCI явно показывает:

```text
Полный режим без проверки портов
```

Это не считается аварией и не отключает детектор. `Сокращённый режим` намеренно не требует `nmap`.

Установка `nmap`:

1. никогда не запускается от одного выбора `Полный режим`;
2. требует отдельного подтверждения родителя;
3. проверяет наличие `opkg` или `apk` и свободное место overlay;
4. обновляет индексы только внутри явно запущенного действия;
5. устанавливает только именованный пакет `nmap`, без массового upgrade;
6. проверяет появление исполняемого файла;
7. публикует ограниченный runtime-статус без секретов.

Заводской минимальный запас свободного места перед установкой — 16 МиБ. Он является защитной границей, а не обещанием одинакового фактического размера пакета на всех ветках OpenWrt.

## Проверки перед релизом

Локальные contract/runtime-тесты не заменяют живой прогон. Перед заявлением готовности требуются:

- сохранение уже настроенного нестандартного часового пояса на реальном роутере;
- первый запуск с пустым system timezone для каждого country profile;
- NTP-синхронизация и расписание после reboot;
- поведение при неверных часах до и после NTP;
- `Полный режим` на роутере без `nmap`;
- успешная и неуспешная установка `nmap` через фактический package manager;
- проверка свободного flash до и после установки;
- подтверждение, что отсутствие `nmap` не останавливает обычное обнаружение устройств.
