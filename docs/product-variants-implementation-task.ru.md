# Задание разработчику: две редакции OpenWrt и два единых Android APK

<!-- §prodvar -->

Развивай `Sheepfold` и `Sheepfold - AI Support` из одного репозитория и одной реализации общего контроля. Не создавай копии firewall, API, расписаний, pairing или Android-навигации.

## Обязательный результат релиза

```text
Sheepfold Standard IPK для OpenWrt 24.10
Sheepfold - AI Support IPK для OpenWrt 24.10
Sheepfold Standard apk-tools v3 APK для OpenWrt 25.12
Sheepfold - AI Support apk-tools v3 APK для OpenWrt 25.12
единый родительский Android APK
единый детский Android APK
```

Android product flavors для AI запрещены. Различие продукта определяется установленной редакцией роутерного пакета и подтверждённой capability роутера. OpenWrt APK и Android APK — разные форматы; запрещено получать OpenWrt APK переименованием или перепаковкой IPK.

## Правила границы

1. Обычный роутерный пакет физически не содержит AI endpoint, handlers, prompts, provider keys, activity collector и AI UI LuCI.
2. AI Support содержит тот же common-код и добавляет AI-only payload.
3. Оба Android APK содержат AI-клиент, но не показывают его без положительного серверного флага.
4. Отсутствующее поле, старый backend, ошибка сети или разбора означают `AI недоступен`.
5. Родительский APK читает `capabilities.aiAssistant` только из авторизованного `/router-info`.
6. Детский APK читает `childAiAvailable`, `childAiAllowed` и `personalGroupRequired` из `/client-status`; backend всё равно повторяет проверку при запросе.
7. Ключи провайдеров хранятся на роутере и никогда не передаются Android.
8. Диагностика, журналы, имена, MAC/IP и семейный контекст не отправляются провайдеру без предпросмотра и явного согласия.
9. ИИ не меняет настройки молча: показать предложение, уточнить понимание, запросить подтверждение, затем вызвать типизированное действие.
10. Обычная сборка не копирует файлы в Downloads. Экспорт выполняется только по прямой просьбе пользователя.
11. Все четыре роутерных release-файла имеют одно внутреннее имя `luci-app-sheepfold-family-internet-control`, но разные имена файлов редакций. Не возвращай взаимный конфликт двух актуальных редакций: переход Standard ↔ AI Support должен быть обновлением одного пакета с сохранением `/etc/config/sheepfold`.
12. При смене редакции на той же версии используй установочный скрипт с резервной копией конфига и принудительной переустановкой через общий package-manager adapter: `opkg` на 24.10 либо `apk` v3 на 25.12. Не удаляй прежний пакет отдельной командой перед загрузкой нового.
13. Старое тестовое имя пакета `luci-app-sheepfold-ai-support` поддерживается только как заменяемое наследие. AI-поля при переходе в Standard остаются неактивными в UCI, чтобы обратный переход не требовал повторной настройки.

## После изменения

1. Запусти точечные Node/shell tests.
2. Собери оба быстрых тестовых IPK через `python scripts/build-test-ipk.py --variant all`.
3. Распакуй их тестом и докажи отсутствие AI-backend в обычном IPK.
4. Собери `assembleDebug` в `android` и `android-child`.
5. Проверь два APK через `scripts/verify-android-variant.py --kind parent|child`.
6. Проверь сценарии: Standard → вкладки нет; AI без ключа → вкладки нет; AI с ключом → вкладка есть; детский доступ без личной группы → баннер без возможности отправки.
7. Запусти полный набор тестов и обнови `docs/current-implementation-status.md`.
8. Запусти `.github/workflows/build-openwrt-packages.yml` и проверь полную SDK-матрицу: две редакции IPK для OpenWrt 24.10 и две редакции apk-tools v3 APK для OpenWrt 25.12, внутренние метаданные, `SHA256SUMS` и build manifest (§owrtci1).
9. На живых тестовых роутерах соответствующих поколений проверь Standard → AI Support → Standard, включая переход на одной версии, сохранность устройств, списков, групп, расписаний, мессенджера и неактивных AI-параметров.

Не останавливайся на визуальном скрытии: capability вычисляет роутер, а каждый AI-запрос заново проходит backend-проверку.
