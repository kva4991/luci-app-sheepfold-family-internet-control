# ИИ-мониторинг безопасности детей

> Статус: **исследовательская идея, не готовая архитектура и не реализовано**. Перед проектированием обязательно свериться с [`ai-assistant-development/`](ai-assistant-development/README.md): нельзя тайно анализировать семью, выдавать вывод модели за факт или создавать психологические ярлыки без явного opt-in, provenance и подтверждения.
>
> **Блокирующее замечание:** описанная ниже слепая передача `alert`/`urgent` прямо родителю не одобрена для реализации. Родитель или опекун может быть источником угрозы, а уведомление способно увеличить опасность. Будущий safety-router может выбрать только безопасного разрешённого семейного/доверенного взрослого для минимального уведомления. Sheepfold никогда не обращается в государственные или экстренные службы сам, а помогает человеку сделать это самостоятельно (§aichild, §aigov0). Этот документ описывает другую исследовательскую идею — анализ сетевых сигналов — и не заменяет безопасный разговор ребёнка с ИИ.

## Цель

Автоматически обнаруживать опасные тенденции в онлайн-поведении детей (игровая зависимость, деструктивные интересы, нездоровые отношения, опасные контакты) и деликатно информировать родителей.

## Когда запускается

- Раз в **3 дня** для каждого ребёнка в личной группе с ролью `child`.
- При накоплении срочных тревожных паттернов — внеплановый запуск.

## Уровни тревоги

Автоматический анализ сетевой активности не может сам по себе подтвердить экстренное событие. Совпадение двух LLM-ответов также не является доказательством. `urgent` из этого модуля означает запрос на дополнительную проверку safety-router, а не готовое решение раскрыть данные. Вызов службы самим Sheepfold запрещён независимо от уровня (§aichild, §aigov0).

| Уровень | Описание | Действие |
|---------|----------|---------|
| `watch` | Слабый сигнал, не достаточно данных | Записать в профиль, отслеживать |
| `alert` | Устойчивый паттерн, требует внимания | Уведомление в мессенджер |
| `urgent` | Возможен серьёзный риск | Передать в safety-router; минимально уведомить безопасного разрешённого семейного/доверенного взрослого либо помочь человеку обратиться за внешней помощью самостоятельно |

## Промпт для ИИ

```
SYSTEM:
Ты — исследовательский AI-классификатор сигналов интернет-безопасности, а не детский психолог. Анализируй только разрешённую структурированную сводку и возвращай кандидаты для последующей проверки человеком.

Правила:
- Не паникуй. Единичные события — не повод для тревоги.
- Паттерн — это устойчивое поведение на протяжении ≥ 5 дней или острые события (контакты с незнакомцами, поиск опасного контента).
- Категории мониторинга: игровая зависимость, ночная активность, деструктивный контент, изоляция, опасные знакомства, суицидальные темы, буллинг.
- Возвращай только JSON.

FORMAT:
{
  "childId": "pg_a1b2c3",
  "overallRisk": "low|medium|high",
  "alerts": [
    {
      "category": "gaming_addiction",
      "level": "watch|alert|urgent",
      "evidence": "кратко — что именно наблюдается",
      "parentMessage": "мягкое сообщение для родителя о чём поговорить",
      "conversationStarter": "конкретная фраза которую родитель может сказать ребёнку"
    }
  ],
  "positiveNotes": "что хорошего замечено за период"
}

USER:
<JSON с активностью ребёнка за период + профиль ребёнка + семейный контекст>
```

## Уведомление родителю

Ранняя идея предполагала при `alert` или `urgent` отправлять в мессенджер (Telegram/VK) сообщение:

```
🔔 Sheepfold: есть кое-что важное про Васю.
Когда будет минута — спросите у ИИ-помощника: «О чём ты хотел поговорить про Васю?»
```

Даже такое сообщение может быть опасно, если получатель является участником угрозы. Поэтому прямой поток считается неподтверждённым и не реализуется до появления safety-router, оценки возможного агрессора и правил выбора безопасного живого получателя. Если родитель безопасен и быстрее всего может спасти ребёнка, safety-router должен уметь уведомить его минимально необходимым сообщением (§aichild).

## Разговор родителя с ИИ-помощником

Когда родитель спрашивает «О чём ты хотел поговорить про Васю?» — ИИ-помощник:
1. Загружает флаг из профиля ребёнка (`pendingParentConversation`).
2. Мягко, без паники, объясняет что заметил.
3. Предлагает конкретные темы для разговора с ребёнком (`conversationStarter`).
4. Снимает флаг `pendingParentConversation` после ответа.
