# IPv6 и совместимость с Podkop

<!-- §ipv6pod -->

## Почему существует настройка

В `Настройки -> Разное -> Совместимость сети` Sheepfold показывает настройку `Выключить IPv6 на роутере`.

По умолчанию она выключена. Для режимов интеграции `Podkop` и `AdGuard Home + Podkop` Sheepfold включает её автоматически и не позволяет случайно снять флаг, пока выбран один из этих режимов.

Это ограничение относится к текущим версиям Podkop, а не считается вечным свойством проекта. В документации Podkop отключение IPv6 рекомендуется для корректной работы клиентского DNS, а полноценная поддержка IPv6 всё ещё указана среди будущих задач:

- https://podkop.net/docs/client-dns/#отключение-ipv6
- https://github.com/itdoginfo/podkop#будущее

Если будущая стабильная версия Podkop официально получит полную поддержку IPv6, автоматическое правило нужно пересмотреть вместе с тестами и подсказкой интерфейса.

## Как применяется

Sheepfold не переписывает чужие секции `/etc/config/network`, `/etc/config/dhcp`, таблицы Podkop или конфигурацию sing-box.

Backend создаёт только принадлежащий Sheepfold файл:

```text
/etc/sysctl.d/99-sheepfold-disable-ipv6.conf
```

Перед первым отключением сохраняются текущие значения `net.ipv6.conf.all.disable_ipv6` и `net.ipv6.conf.default.disable_ipv6`. При ручном выключении функции или уходе с Podkop прежние значения восстанавливаются. Благодаря этому Sheepfold не предполагает, что до установки IPv6 обязательно был включён.

После импорта резервной копии backend сразу применяет восстановленное значение, а не ждёт перезагрузки роутера. При обычном обновлении IPK состояние IPv6 не переключается между старым и новым пакетом. При полном удалении пакета Sheepfold возвращает сохранённые kernel-значения и удаляет только свой sysctl-файл и служебный снимок. `/etc/config/sheepfold`, устройства и остальные пользовательские настройки при этом не очищаются.

Если восстановить kernel-значения при удалении не удалось, удаление пакета продолжается, но в системный журнал и консоль записывается предупреждение. Это позволяет завершить удаление и одновременно не скрывает оставшуюся настройку от владельца роутера.

## Источник настройки

- `router_ipv6_disabled=0|1` — выбранное состояние;
- `router_ipv6_mode_source=default|manual|auto_podkop` — причина выбора.

Если значение было включено автоматически из-за Podkop, уход на режим без Podkop возвращает значение `0`. Ручное включение без Podkop сохраняется.

## Проверка на живом роутере

1. Проверить сохранение настройки и отсутствие незавершённых LuCI-изменений.
2. Проверить `sysctl net.ipv6.conf.all.disable_ipv6` и `sysctl net.ipv6.conf.default.disable_ipv6`.
3. Проверить LuCI, DNS, Sheepfold, AdGuard Home и Podkop.
4. Проверить восстановление исходных значений после ухода с Podkop.
5. Проверить повторное применение после перезагрузки роутера.

Клиентам может понадобиться переподключение к сети, чтобы перестать использовать ранее выданные IPv6-настройки.
