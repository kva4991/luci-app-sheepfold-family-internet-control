### 2. Именование по слоям

Стиль проекта зависит от слоя (§pm3kq7r). В JS/Kotlin/JSON-моделях используем camelCase, потому что это привычный стиль для LuCI frontend, Android и HTTP API. В OpenWrt/UCI, shell-скриптах и Makefile сохраняем привычный для OpenWrt стиль: `snake_case` для UCI-опций и shell-переменных, верхний регистр для Makefile-переменных. Не переименовывать существующие UCI-поля без миграции.

Новые локальные переменные и приватные поля должны иметь понятное человеку имя длиной не более 15 символов. Не сокращать их до нечитаемого набора букв ради лимита: использовать устойчивые термины проекта (`deviceMac`, `routerAddr`, `adminList`) или упростить слишком широкий блок кода. Внешние контракты, generated code, constants и platform identifiers под этот числовой лимит не попадают.

Примеры правильных имён:
- JS/Kotlin: `deviceId`, `displayName`, `personalGroupId`, `ownerChangedAt`, `activeMessenger`, `adminDevice`;
- UCI/shell: `device_id`, `display_name`, `personal_group_id`, `active_messenger`, `admin_device`;
- Makefile: `PKG_RELEASE`, `SHEEPFOLD_UI_ASSET_VERSION`.

### 3. ИИ-промпты: тонкие места

**ai-device-ownership:**
- Промпт не учитывает случай когда у человека нет ни одного устройства в личной группе ещё — тогда ИИ не знает имён членов семьи. Нужно передавать имена из `displayName` всех `personal_group`-секций даже если они пустые.
- Порог 85% субъективен (§vq8xb5d). Рассмотреть настройку через UCI: `sheepfold.global.aiOwnershipAutoAssignThreshold`.

**ai-family-profiles:**
- Нет защиты от «галлюцинаций» ИИ — профиль может содержать выдуманные факты. Нужна валидация: ИИ не должен добавлять информацию которой нет в переданных данных активности.
- Поле `approachHints` — самое ценное. Не допускать его удаления при обновлении если нет новых данных.

**ai-child-safety:**
- `conversationStarter` — рискованное поле: ИИ может предложить неуместную фразу. Нужен human-review или минимальная модерация перед отправкой.
- Уровень `urgent` должен требовать подтверждения от второго ИИ-запроса (двойная проверка) перед отправкой уведомления (§kj2n6wt).

### 4. Вкладка «ИИ-помощник» в APK

Вкладка отображается только если роутер сообщает, что на нём настроен хотя бы один ИИ-провайдер. Android не проверяет provider API keys напрямую и не хранит их у себя: он читает безопасный флаг/список провайдеров из router-side discovery/status API и отправляет запросы только через Sheepfold backend (§xaji0y6, §dpbhsah, §c9rf1pe).

## Неоднозначные места в архитектуре

- **Приватность профилей**: семейные профили хранятся на роутере в `/etc/sheepfold/`. При сбросе роутера — удаляются. Нужен ли backup в облако? Пока нет — это намеренно (конфиденциальность).
- **Какой ИИ использовать**: OpenAI GPT-4o-mini дешевле, но хуже понимает русский. YandexGPT лучше для русского контекста. Gemini — хороший баланс. Провайдер должен быть выбором пользователя (§t3w5uzb).
- **Частота профилирования**: 7 дней — компромисс. Для семей с маленькими детьми может быть мало. Рассмотреть настройку через UCI.
