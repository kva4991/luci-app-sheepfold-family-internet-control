# Прямое задание для ИИ-разработчика

Ты работаешь над Sheepfold Family Internet Control.

Твоя задача — превратить текущий каркас в рабочий self-hosted продукт для OpenWRT-роутера с LuCI-интерфейсом, Android-приложением для родителей-администраторов и опциональным управлением через Telegram или VK. VK — вариант по умолчанию при первичной настройке мессенджера. MAX можно оставить экспериментальным адаптером.

Перед началом работы прочитай:

- `AGENTS.md`;
- `docs/agent-playbook.ru.md`;
- `docs/product-requirements.md`;
- `docs/security.md`;
- `docs/privacy.ru.md`;
- `docs/messaging.ru.md`;
- `docs/schedules.ru.md`;
- `docs/integrations.md`;
- `docs/domain-allowlist.ru.md`;
- `docs/podkop-luci-notes.ru.md`;
- `docs/ai-context-sharing.ru.md`.

## Главный сценарий

Родитель устанавливает Sheepfold на OpenWRT-роутер, открывает LuCI, видит все домашние устройства, настраивает белый список, чёрный список, расписания, временный доступ и аварийно-полезные сайты.

Android-приложение `Овчарня` дублирует основные родительские действия. Оно устанавливается только на телефоны родителей-администраторов.

Вне дома родитель может управлять короткими командами через один выбранный мессенджер: Telegram, VK или экспериментальный MAX.

## Что сделать в первую очередь

1. Реализовать backend/API на роутере.
2. Реализовать LuCI-интерфейс с реальными вкладками:
   - белый список;
   - чёрный список;
   - все устройства;
   - расписания;
   - аварийно-полезные сайты;
   - Wi-Fi 2.4/5 GHz;
   - интеграции;
   - администраторы;
   - бот/мессенджер;
   - настройки;
   - журнал.
3. Реализовать обнаружение устройств из OpenWRT: DHCP leases, static leases, ARP/neighbor data.
4. Синхронизировать имена устройств и постоянные IP с OpenWRT DHCP static leases.
5. Реализовать правила firewall через `firewall4` / `nftables`.
6. Реализовать импорт/экспорт настроек, включая клиентов.
7. Реализовать безопасное обновление и удаление.
8. Реализовать Telegram-адаптер через outbound HTTPS long polling.
9. Реализовать VK-адаптер через тот же внутренний интерфейс мессенджеров.
10. MAX оставить экспериментальным и не блокировать им первый стабильный релиз.
11. Реализовать единое версионирование LuCI JS/CSS/static ассетов для сброса браузерного кэша после обновлений.
12. Реализовать локальное сопряжение Android-приложения через QR/ручные настройки из LuCI.
13. После стабильного роутерного ядра проектировать Android-приложение.

## Жёсткие правила

- Устройство не может одновременно быть в белом и чёрном списке.
- Чёрный список всегда сильнее любых временных разрешений.
- Белый список не блокируется расписаниями и глобальной блокировкой.
- Глобальная кнопка "Выключить интернет" блокирует всех, кроме белого списка.
- Поведение новых устройств настраивается: `allow` по умолчанию или `restrict_until_configured`.
- Нужны группы устройств: дети, родители, телевизоры/медиа, гости/пользовательские группы.
- Устройства из чёрного списка не должны открывать LuCI, SSH и Sheepfold API.
- Аварийно-полезные сайты могут быть доступны даже чёрному списку только через отдельную явную настройку, но LuCI/SSH/API всё равно закрыты.
- Заблокированные не по чёрному списку устройства могут видеть LuCI/API только если включена соответствующая настройка.
- Аварийно-полезные сайты выключены по умолчанию и включаются родителем.
- Не добавлять `yandex.ru`, маркетплейсы, супераппы, доставку еды, видео, игры, соцсети и app stores в дефолтный список.
- Яндекс Go / Яндекс Такси не предлагать по умолчанию.
- Android-приложение не ставится на телефоны детей.
- Не вводить облако разработчика.
- Не добавлять сценарий полного управления через WireGuard/VPN-туннель к роутеру.
- Не хранить пароль root роутера в Android.
- Не отправлять MAC/IP/имена детей/устройств/семейный контекст/логи ИИ-провайдеру без отдельного preview и подтверждения.
- Не обещать Android-приложению автоматическое отключение случайного/private MAC для Wi-Fi: приложение должно проверять, объяснять и вести родителя в системные настройки.
- QR-код сопряжения не должен содержать root-пароль роутера, LuCI session cookies, токены ботов, AI-ключи или другие посторонние секреты.
- При первом открытии Sheepfold в LuCI проверять, задан ли root-пароль OpenWRT. Если не задан — не пускать в настройки Sheepfold и отправлять пользователя на страницу установки пароля роутера.
- В LuCI показывать родителю одну понятную кнопку `Сохранить`; отдельную кнопку `Применить` скрывать/объединять с сохранением.
- В белом списке реализовать быстрое добавление: окно с QR-кодом Wi-Fi, 30-секундным таймером подключения и списком устройств, подключившихся после запуска окна. У каждого кандидата должна быть кнопка `Добавить`; автоматически добавлять всех нельзя.

## OpenWRT

Цель — современные OpenWRT с `firewall4` / `nftables`.

Не реализовывать `firewall3` / `iptables`, если владелец проекта явно не попросит позже.

Установщик должен:

- требовать согласия с пользовательским соглашением;
- определить OpenWRT;
- определить наличие AdGuard Home и Podkop;
- выбрать рекомендуемый `integration_mode`;
- не перезаписывать настройки AdGuard Home/Podkop без отдельного подтверждения.

## LuCI

LuCI — основной интерфейс настройки.

Он должен быть понятным для родителя, а не только для сетевого администратора.

Все опасные действия требуют подтверждения:

- перезагрузка роутера;
- обновление;
- импорт настроек;
- очистка журнала;
- массовая блокировка;
- смена Wi-Fi имени/пароля;
- смена активного мессенджера;
- добавление администратора.
- экспорт полного бэкапа с секретами;

Во вкладке `Администраторы` нужен один владелец по умолчанию и возможность создавать дополнительных администраторов. У каждого администратора есть уникальное имя, логин, роль и пароль. Пароли хранить только в виде хеша. Последнего владельца нельзя удалить или понизить.

В списке устройств любое устройство можно сделать админским. При этом LuCI должен спросить, какому администратору оно принадлежит. Для админских устройств в таблице показывать особый локальный значок по смыслу FontAwesome `laptop-mobile` и кнопку `Сопряжение`. Нажатие открывает окно с QR-кодом для Android и текстовыми ручными настройками рядом: адрес роутера/API, логин или ID администратора, одноразовый код/токен, срок действия, подсказка по Wi-Fi MAC. Токен короткоживущий, одноразовый, привязан к администратору и устройству, хранится на роутере только как хеш/невосстановимый секрет.

LuCI-ассеты должны использовать один cache-busting version suffix. Источник версии — `PKG_VERSION-PKG_RELEASE` OpenWRT-пакета, runtime-значение можно хранить как `ui_asset_version`. Все Sheepfold JS/CSS/static URL должны получать один и тот же suffix вида `?v=0.1.0-1`. Не хардкодить отдельные версии в разных frontend-файлах.

Рабочую LuCI-архитектуру делай по Podkop-подобной схеме: небольшой entrypoint, `form.Map("sheepfold")`, `tabbed = true`, отдельный модуль на каждую вкладку, отдельный слой backend methods и строгий rpcd ACL. Не разращивать один огромный `overview.js`.

## Android

Минимум: Android 9.0 / API 28.

Первый запуск:

1. Показать пользовательское соглашение и чекбокс согласия.
2. Подключиться к домашнему Wi-Fi и выполнить сопряжение с роутером через QR-код из LuCI или ручной ввод настроек.
3. Проверить, что телефон виден роутеру под настоящим MAC-адресом устройства. Если включён случайный/private MAC для домашней сети, объяснить проблему и открыть/подсказать системные настройки Wi-Fi. Автоматическое переключение не обещать. Пока пользователь не переключил сеть на настоящий MAC и роутер это не подтвердил, дальше не идти.
4. Настроить локальную защиту приложения.
5. Рекомендовать пароль/PIN, а не биометрию.

Полный интерфейс работает только локально с роутером в домашней сети. Не проектировать и не обещать полный удалённый доступ через WireGuard, VPN-туннель или другой туннель к роутеру. Удалённое управление вне дома — только короткими подтверждаемыми командами через Telegram/VK-бота или экспериментальный MAX-адаптер.

## Бот

На роутере одновременно активен только один мессенджер: Telegram, VK или экспериментальный MAX.

При первичной настройке мессенджера по умолчанию предлагать VK. При этом установленный конфиг не должен делать мессенджер активным, пока родитель не ввёл токен/данные доступа и не привязал хотя бы одного разрешённого администратора.

Telegram:

- реализовать через `getUpdates` long polling;
- не открывать входящие порты;
- хранить offset локально;
- принимать команды только от разрешённых администраторов.

VK:

- использовать тот же внутренний интерфейс адаптеров, что и Telegram;
- принимать команды только от разрешённых администраторов;
- не открывать роутер наружу, если доступен исходящий polling/long-poll сценарий.

MAX:

- держать как экспериментальный адаптер;
- не делать обязательным для первого релиза;
- не включать по умолчанию.

## Экспорт

Обычный экспорт по умолчанию — читаемый JSON/архив без секретов.

Если пользователь хочет полный бэкап с токенами, API-ключами и другими секретами, нужен зашифрованный экспорт с паролем.

Офлайн-устройства чистить через настраиваемое число дней без активности. Дефолт: `90`.

## Заглушка

Для заблокированных устройств нужна локальная HTML-страница на роутере.

Текст заглушки должен настраиваться родителем.

## Логи

Дефолт:

```text
retention: 3d
max size: 1024 KB
masking: balanced
```

Не писать секреты, пароли, токены, session cookies, полные переписки, полные AI-запросы, историю сайтов, DNS-query-log всех устройств, банковские и медицинские данные.

## Интеграции

Целевая цепочка:

```text
Sheepfold -> AdGuard Home -> Podkop
```

Поддерживаемые режимы:

```text
none
adguard
podkop
adguard_podkop
```

AdGuard Home и Podkop не взаимоисключают друг друга.

- изменения соответствуют `AGENTS.md`;
