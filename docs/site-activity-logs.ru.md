# Журнал интернет-активности устройств

Журнал интернет-активности — отдельная чувствительная функция Sheepfold. Это не обычный административный журнал и не скрытая слежка.

Цель функции: помочь родителю понять контекст проблемы и получить спокойный разбор от ИИ-помощника, а не дать повод для мгновенного скандала с ребёнком.

## Основной принцип

Мы исходим из того, что у ребёнка могут быть “левые телефоны” и подменённые данные. Поэтому Sheepfold не доверяет приложению на телефоне ребёнка и вообще не требует установки APK на детские устройства.

Источники данных только со стороны роутера и локальной инфраструктуры:

- DHCP/ARP/neigh;
- DNS-данные, если они проходят через роутер или AdGuard Home;
- сетевые подключения и домены, если их можно увидеть законным и технически корректным способом;
- метаданные из разрешённых внешних источников только после явного включения.

Если роутер не видит данные, Sheepfold должен писать “недоступно”, а не додумывать.

## Для кого собирать

Журнал интернет-активности может собираться только для обычных устройств.

Не собирать:

- для устройств администраторов;
- для устройств из белого списка;
- для устройств из чёрного списка;
- для устройств, где родитель явно выключил журнал;
- если глобальная настройка истории выключена.

В таблице “Все устройства” для устройства с включённой записью показывать плашку `Журнал`.

В настройках группы:

```text
Включить журнал активности для всех устройств этой группы = да/нет
```

По умолчанию: `нет`.

В настройках устройства:

```text
Включить журнал активности для этого устройства = да/нет
```

По умолчанию: `нет`.

## Что показывать родителю

В LuCI и APK нужно показывать:

- что журнал для устройства включён;
- примерный размер журнала;
- период хранения;
- кнопку очистки;
- кнопку экспорта с выбором периода: за последний час, за неделю, с-по, за всё время.

Сырой журнал сайтов по умолчанию не показывать. Родитель может попросить ИИ обработать журнал, но перед отправкой должен увидеть отдельное подтверждение: какие поля уйдут ИИ-провайдеру и за какой период.

## Что можно хранить

Минимальный допустимый набор:

- ID устройства Sheepfold;
- локальное имя устройства;
- домен или hostname;
- категория, если определена;
- время первого и последнего обращения;
- счётчик обращений;
- примерная длительность активности, если её можно оценить без глубокого анализа трафика;
- источник данных: router DNS, AdGuard Home, connection metadata, manual import.

Дополнительные метаданные по видео/страницам допустимы только если они получены корректным способом и включены родителем:

- заголовок видео/страницы;
- краткое описание;
- возрастной рейтинг, если доступен;
- признаки комментариев или описания, если источник это разрешает и технически возможно.

Важно: DNS сам по себе не даёт заголовки видео, описания и комментарии. Для этого нужен отдельный механизм метаданных, API платформы, прокси-режим или ручная передача ссылки родителем. Если данных нет, ИИ не должен делать вид, что видел содержание.

## Что нельзя хранить

Не хранить:

- полный URL с приватными параметрами;
- поисковые запросы;
- POST/PUT/PATCH/DELETE body;
- cookies, session IDs, authorization headers;
- содержимое страниц;
- банковские, медицинские и личные формы;
- приватные переписки;
- данные админских устройств, белого списка и чёрного списка.

## Как ИИ должен анализировать

ИИ-помощник не должен перечислять родителю сырой список сайтов и заголовков в стиле “ребёнок ходил туда-то”. Это может спровоцировать злость и прямой конфликт.

Правильный ответ ИИ:

- описать спокойные паттерны: ночная активность, резкий рост времени, обход фильтров, опасные темы;
- отделить факт от предположения;
- предложить темы для разговора с ребёнком;
- дать фразы, которыми можно начать разговор без нападения;
- сказать, чего не делать: не врываться с обвинениями, не стыдить, не читать нотацию, не показывать ребёнку “досье”;
- при признаках самоповреждения, суицида, наркотиков, насилия, террористического вовлечения или сексуальной эксплуатации рекомендовать немедленную помощь взрослых и профильных/экстренных служб по месту нахождения семьи.

ИИ должен учитывать не только заголовок, но и доступные описание/комментарии/категорию, потому что безобидный заголовок может скрывать опасный контент. Если такие данные недоступны, ИИ обязан явно сказать об ограничении анализа.

## Хранение

Рекомендуемые настройки:

- по умолчанию: выключено;
- дефолтный срок при включении: 24 часа;
- максимум без отдельного подтверждения: 7 дней;
- хранить в RAM или в явно выбранном файле с предупреждением о flash-памяти;
- отдельный лимит размера;
- отдельная кнопка очистки.

## Экспорт

При нажатии `Экспорт` открывать модалку:

- за последний час;
- за неделю;
- с - по;
- за всё время;
- кнопка `Экспортировать`.

По умолчанию экспорт должен маскировать MAC/IP/имена, если пользователь явно не выбрал другой режим.
