# Карта тегов (§-теги)

В документации и исходном коде Sheepfold используется система перекрёстных тегов-сносок.

---

## Что такое §-тег

Тег — это типографский символ `§` (параграф) и 7-символьный буквенно-цифровой идентификатор без пробела, заключённый в скобки:

```
(§xaji0y6)
```

**Тег — это как книжная сноска:** он ставится в месте, где уместно дать дополнительный контекст,
но писать его прямо там было бы слишком длинно или не по месту.
Расшифровка тега — в этом файле, в таблице ниже.

Тег **не заменяет** обычный комментарий в коде или описание в документации.
Он дополняет: «здесь есть нюанс → смотри в карте тегов».

---

## Правила расстановки

### В Markdown-документации

- Тег ставится **в тексте абзаца** в скобках: `(§xaji0y6)`.
- Тег **нельзя ставить в заголовках** (`## Заголовок`).
- Тег можно ставить в конце предложения, в списке, в таблице, в блоке кода только внутри комментария.

Пример:
```markdown
API-ключи хранятся через Android Keystore (§dpbhsah), никогда не передаются в UCI на роутер.
```

### В shell-скриптах

- Тег ставится **только внутри существующего комментария**, в конце строки:

```sh
# Проверяем наличие WAN-интерфейса (§f1t2tal)
if ubus call network.interface.wan status ...
```

- **Не добавлять отдельную строку комментария только ради тега** — тег должен дополнять уже написанный комментарий.

### В Kotlin/Java

```kotlin
// Запрос идёт через роутерный API, не напрямую к LLM-провайдеру (§xaji0y6)
val response = sheepfoldApi.aiAssistant(request)
```

### В Lua

```lua
-- UCI-секции должны иметь уникальные имена (§w2pcn9t)
local cfg = uci:get("sheepfold", "global")
```

---

## Как найти тег

Поиск по всему репозиторию:
```bash
grep -r '§' docs/ usr/
```

Поиск конкретного тега:
```bash
grep -r '§xaji0y6' .
```

---

## Как добавить новый тег

1. Сгенерировать 7-символьный id из `[a-z0-9]`, убедиться что такого нет в таблице ниже.
2. Добавить строку в таблицу ниже.
3. Расставить тег в нужных местах документации и/или кода.

---

## Карта тегов

| Тег | Имя | Тип | Описание / Что означает нахождение тега в тексте |
|-----|-----|-----|--------------------------------------------------|
| `§xaji0y6` | AI_NO_DIRECT_LLM | архитектура | Android-приложение **не ходит напрямую** к LLM-провайдеру (DeepSeek, Gemini и т.п.) — все запросы идут через `/cgi-bin/sheepfold-api/ai-assistant` на роутере. Тег в коде означает: именно здесь реализуется эта прослойка или явно полагается на неё. |
| `§dpbhsah` | AI_KEYSTORE | безопасность | API-ключи LLM-провайдеров хранятся через Android Keystore и **никогда не передаются** в UCI на роутер, если не реализован явный server-side proxy mode. Тег в коде означает: здесь происходит работа с хранилищем ключей. |
| `§xthv3a3` | AI_CONTEXT_CONFIRM | UX/безопасность | Перед отправкой любого расширенного контекста из Sheepfold к ИИ-провайдеру обязательно явное подтверждение пользователя с preview содержимого. Тег означает: здесь реализован или проверяется этот экран/диалог подтверждения. |
| `§zmf8mdd` | AI_ROUTER_SNAPSHOT | API | Диагностический снимок роутера для ИИ: версия, модель, статус сети, ping, модули Wi-Fi без паролей, uptime/load/memory. Состав полей описан в `ai-context-sharing.ru.md`. Тег означает: здесь формируется или используется этот снимок. |
| `§4v30t9n` | AI_FORBIDDEN_FIELDS | безопасность | Полный список полей, **запрещённых к автоматической отправке** ИИ: MAC, IP, имена детей, токены, API-ключи, пароли, журналы, DNS-запросы и т.д. Тег означает: здесь применяется фильтрация или проверяется что запрещённые поля не попали в payload. |
| `§t3w5uzb` | AI_PROVIDERS_COUNTRY | конфигурация | Список доступных LLM-провайдеров определяется **country profile** роутера и может меняться. Тег означает: здесь происходит выбор или фильтрация провайдеров по стране. |
| `§ikcidkw` | AI_MODEL_CONFIG | конфигурация | Имена моделей LLM **не зашиты в код** — хранятся в конфигурации провайдера. Тег означает: здесь читается или записывается model ID, и важно не хардкодить его. |
| `§nnhj7xv` | AI_PROMPT_REVIEW | процесс | Промпт ИИ-помощника перед production-релизом **должен пройти ревью семейного психолога**. Тег в документации означает: это место или документ связан с промптом и подлежит такому ревью. |
| `§g0fn9xu` | AI_ROLE_CHILD | безопасность/UX | Запросы от ребёнка идут с явной пометкой `role=child`. ИИ в ответах не предлагает действий уровня родителя, не раскрывает лишних деталей, использует мягкий тон. Тег означает: здесь реализована или проверяется ролевая логика для ребёнка. |
| `§y41iblj` | AI_ENDPOINT_STUB | статус реализации | `/cgi-bin/sheepfold-api/ai-assistant` сейчас **заглушка** — полный authenticated API и role-based auth не завершены. Тег означает: этот код или тест опирается на заглушку и нуждается в обновлении при завершении auth-слоя. |
| `§h75lxo6` | API_BEARER_TOKEN | безопасность | Каждый запрос к Sheepfold API проверяет Bearer-токен администратора. Токены хранятся как HMAC-хэши, plaintext не хранится. Тег означает: здесь происходит проверка токена или важно помнить что endpoint требует авторизации. |
| `§qjiujv6` | API_CONFIRM_TOKEN | безопасность | Опасные действия (reboot, update, global-block, clear-log) требуют отдельный `confirm`-токен помимо Bearer. Тег означает: этот endpoint или вызов относится к «опасным» и требует двойного подтверждения. |
| `§oh9sdbd` | API_NO_SECRETS_RESPONSE | безопасность | `router-info` и другие публичные endpoints **никогда не включают** в ответ: пароли Wi-Fi, токены ботов/AI, имена детей, MAC-адреса устройств, списки клиентов. Тег означает: здесь формируется ответ API и нужно убедиться что секреты не утекли. |
| `§w2pcn9t` | UCI_UNIQUE_SECTION | конфигурация | В одном UCI-файле не должно быть двух секций с одинаковым именем. Использовать уникальные имена: `messenger_global`, `pairing_global`, а не повторно `global`. Тег означает: здесь создаётся или читается UCI-секция, и имя должно быть уникальным. |
| `§84azytj` | ACCESS_PRIORITY_ORDER | логика доступа | Полный порядок приоритетов правил доступа: blocklist → allowlist → no-restrictions → global-block → temp-access → schedule → default. Тег означает: здесь реализуется или проверяется одно из звеньев этой цепочки, и важно помнить общий порядок. |
| `§xepq85j` | ACCESS_BLOCKLIST_FINAL | логика доступа | Blocklist **не переопределяется ничем** — ни allowlist, ни temp-access, ни no-restrictions. Тег означает: здесь есть логика, которая могла бы ошибочно «пробить» blocklist. |
| `§sg65kxv` | ACCESS_ALLOWLIST_BYPASS | логика доступа | Allowlist обходит расписания, временный доступ и глобальную блокировку, но **не** blocklist. Тег означает: здесь реализован или проверяется bypass для allowlist-устройств. |
| `§f1t2tal` | DIAG_CHAIN | диагностика | Цепочка диагностики интернет-соединения: WAN link → default route → DNS (ya.ru/gosuslugi.ru) → HTTP(S). Тег означает: здесь реализован один из шагов цепочки или логика перехода между статусами `offline`/`limited`/`online`. |
| `§a753lc5` | DIAG_NO_GOOGLE_DNS | диагностика | Диагностика **не должна** использовать `1.1.1.1`, `8.8.8.8`, `google.com` как единственные цели — они могут быть заблокированы в России. Тег означает: здесь выбираются цели пинга/DNS-проверки, и нужно убедиться что они подходят для РФ. |
| `§8drc11e` | LUCI_EXEC_ONLY | безопасность | LuCI вызывает только `/usr/libexec/sheepfold/sheepfold-router-control` и **не строит shell-команды динамически** из пользовательского ввода. Тег означает: это точка вызова из LuCI — важно убедиться что нет прямой конкатенации shell-строк из ввода. |
| `§rtj5pht` | LOG_RAM_MASKED | приватность | Журнал хранится в RAM (`/tmp/sheepfold/events.log`). При экспорте MAC-адреса и IP маскируются. Тег означает: здесь происходит запись в журнал или его экспорт — нужно убедиться в маскировке. |
