# Передача контекста ИИ-помощнику

> Статус: backend умеет по явным флагам добавить обезличенную диагностику и журнал, но экран preview и отдельные переключатели в родительском APK ещё не реализованы. Текущий UI отправляет только текст родителя (§aiimpl1).

ИИ-помощник по умолчанию должен отправлять провайдеру только текст, который родитель сам написал в чате.

Даже введённый пользователем текст перед внешней отправкой проходит локальное удаление секретов и прямых технических идентификаторов. Если в нём содержатся семейные сведения, имена, здоровье, религия, точные события или иные чувствительные категории, приложение должно показать замаскированный preview. Проверка уже после внешней отправки не считается защитой (§aimask1).

Если родитель хочет получить более точный совет, приложение может предложить добавить контекст из Sheepfold. Это должно быть отдельное действие с явным подтверждением (§xthv3a3).

Если роутеру не хватает ресурсов, разрешённый контекст может быть обработан необязательным домашним или внешним worker только по правилам [`ai-assistant-development/privacy-proxy-and-external-compute.ru.md`](ai-assistant-development/privacy-proxy-and-external-compute.ru.md). Получатель и место выполнения показываются до отправки. Псевдонимизированные данные не называются полностью анонимными (§aiexec1).

## Как реализовать

В Android-приложении нужен экран перед отправкой расширенного запроса:

```text
Добавить контекст из Sheepfold?
```

На экране показываются переключатели:

- возраст ребёнка;
- общая цель родителя;
- текущий стиль ограничений;
- расписание без имён устройств;
- категории правил без MAC/IP (§4v30t9n);
- обезличенный список типов устройств;
- последние административные действия без технических идентификаторов.
- диагностический снимок роутера из `/cgi-bin/sheepfold-api/router-info` (§zmf8mdd): версия Sheepfold, модель роутера, версия OpenWRT, статус интернета, ping до `ya.ru`, Wi-Fi-модули без паролей, LAN-порты, наличие Podkop/AdGuard Home, uptime/load/memory.

По умолчанию все переключатели выключены, кроме явно безопасных подсказок, если пользователь уже включил режим персонализации.

## Диагностика роутера для ИИ

Android-приложение может предложить отправить ИИ-помощнику технический снимок вкладки `Информация` (§zmf8mdd), если родитель спрашивает что-то вроде:

```text
почему у меня плохо работает интернет?
помоги понять, правильно ли настроен роутер
почему не виден Podkop или AdGuard Home
```

Перед отправкой APK обязан показать preview из поля `aiContextPreview` и явно спросить подтверждение (§xthv3a3).

Этот диагностический снимок не должен содержать (§4v30t9n):

- пароли Wi-Fi;
- токены ботов;
- API-ключи;
- session cookies;
- root-пароль роутера;
- MAC-адреса клиентов;
- имена детей;
- списки устройств;
- журналы действий или посещений.

## Запрещено отправлять автоматически

Список запрещённых к автоматической отправке полей (§4v30t9n):

- MAC-адреса;
- IP-адреса;
- имена детей;
- точные названия устройств;
- chat ID / user ID мессенджеров;
- токены;
- API-ключи;
- полный журнал действий;
- историю сайтов;
- DNS-запросы;
- приватные заметки родителя;
- банковские, медицинские и другие чувствительные сведения.

## Рекомендуемый формат

Перед отправкой приложение формирует `context_preview` — текст, который родитель видит до отправки.

Пример:

```text
Контекст для ИИ:
- ребёнок: возраст 10 лет;
- цель: меньше конфликтов из-за игр вечером;
- текущие правила: интернет отключается после 21:00;
- устройство: смартфон ребёнка, без MAC/IP;
- недавнее действие: вчера был выдан временный доступ на 30 минут.
```

Родитель должен нажать (§xthv3a3):

```text
Отправить этот контекст
```

Без этого подтверждения отправляется только обычный текст сообщения.
