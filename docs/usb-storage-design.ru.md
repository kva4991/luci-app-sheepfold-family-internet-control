# USB-хранилище для приватных архивов

## Статус реализации

В текущей ветке реализован CLI backend:

```text
/usr/libexec/sheepfold/sheepfold-usb-storage
```

CGI endpoint `/usb-storage` и отдельный LuCI-таб пока **не реализованы**. Описанные ниже команды нельзя считать доступными через веб-интерфейс до появления отдельного аутентифицированного API, ACL и подтверждений опасных действий.

## Модель хранения

Активные приватные логи находятся в RAM. Команда `archive-push` переносит готовые `.tar` или `.tar.gz` из:

```text
/tmp/sheepfold/.private/archive
```

в:

```text
/mnt/sheepfold-usb/sheepfold/logs
```

Шифрование включается настройкой `sheepfold.usb.encrypt=1` и работает fail-closed: при отсутствии OpenSSL либо ошибке шифрования исходный архив остаётся в RAM, а незашифрованный файл с расширением `.enc` не создаётся.

## Ключ шифрования

Ключ выводится из двух случайных компонентов:

```text
/etc/sheepfold/.kp
/etc/sheepfold/.app_salt
```

Оба файла:

- создаются лениво и атомарно;
- содержат по 32 случайных байта в hex-представлении;
- имеют права `0600`;
- находятся в каталоге с правами `0700`;
- не встраиваются в пакет и не меняются при обычном обновлении.

Итоговый ключ:

```text
SHA-256(.kp + ":" + .app_salt)
```

Шифрование:

```text
openssl enc -aes-256-cbc -pbkdf2 -iter 100000
```

На USB не записывается ни настоящий ключ, ни ложная «приманка». Обманные файлы не являются механизмом безопасности и были удалены.

> При сбросе или замене роутера без резервной копии обоих компонентов старые архивы станут нерасшифровываемыми. До выпуска пользовательского USB-интерфейса необходимо спроектировать безопасный экспорт и восстановление ключевого материала.

## Защита разрушительных операций

Backend не доверяет пути `/dev/...`, полученному из UI или ручного вызова.

Перед `format`, `mount`, `setup-swap` и `setup-extroot` проверяется:

1. путь действительно указывает на block device;
2. тип имени поддерживается (`sdX`, `sdXn`, съёмный `mmcblkN`);
3. sysfs помечает носитель как removable либо показывает USB-шину;
4. тот же физический накопитель не используется для `/`, `/overlay` или swap системы.

Если проверка не пройдена, команда возвращает `unsafe_device` и ничего не форматирует.

## CLI

```bash
sheepfold-usb-storage scan
sheepfold-usb-storage info /dev/sda1
sheepfold-usb-storage format /dev/sda1
sheepfold-usb-storage mount /dev/sda1
sheepfold-usb-storage umount
sheepfold-usb-storage status
sheepfold-usb-storage archive-push
sheepfold-usb-storage setup-swap /dev/sda1
sheepfold-usb-storage setup-extroot /dev/sda1
```

`format`, `setup-swap` и `setup-extroot` являются опасными административными операциями. Будущий LuCI/API слой обязан:

- требовать Bearer/LuCI-аутентификацию;
- показывать модель, размер и точный путь носителя;
- требовать отдельное подтверждение;
- не принимать произвольный путь без повторной backend-проверки;
- журналировать результат без секретов.

## UCI

Предполагаемая секция:

```text
config usb 'usb'
    option device  '/dev/sda1'
    option enabled '0'
    option role    'logs_only'
    option encrypt '1'
```

При отсутствии секции backend использует безопасные значения по умолчанию и ничего автоматически не монтирует.

## Ограничение места

| Роль | Максимальная доля для логов |
|---|---:|
| `logs_only` | 100% |
| `swap_logs` | 70% |
| `extroot_logs` | 70% |

При достижении лимита удаляются самые старые архивы, пока использование снова не станет ниже порога либо пока каталог не опустеет.

## Зависимости

| Возможность | Пакет |
|---|---|
| ext4 | `e2fsprogs`, `kmod-fs-ext4` |
| f2fs | `f2fs-tools`, `kmod-fs-f2fs` |
| extroot | `block-mount` |
| шифрование | `openssl-util` или совместимый пакет OpenSSL |
| USB storage | подходящие kernel-модули конкретной сборки OpenWrt |

OpenSSL проверяется реальной пробной операцией, а не только наличием бинарного файла.

## Автоматический запуск

В репозитории присутствует пример задания `root/etc/cron.d/sheepfold-usb`. Для целевой сборки OpenWrt необходимо отдельно подтвердить поддержку `/etc/cron.d`; BusyBox cron часто использует `/etc/crontabs/root`. До такой проверки автоматический `archive-push` нельзя считать гарантированным.
