# USB-хранилище для логов: дизайн

## Концепция

Флешка — это только **хранилище архивов**. Активная запись логов всегда идёт в RAM.
Ночью (02:05) rotate упаковывает день в `.tar.gz`, затем (02:10) `archive-push`
перемещает архив на флешку, шифруя его перед записью.

---

## Правила заполнения

| Роль флешки | Макс % для логов | Остаток |
|---|---|---|
| `logs_only` | 100% | — |
| `swap_logs` (флешка + подкачка) | 70% | подкачка, другое |
| `extroot_logs` (флешка + /overlay) | 70% | пакеты OpenWRT |

При достижении лимита: удаляется самый старый архив.

---

## Шифрование архивов

- Инструмент: `openssl enc -aes-256-cbc -pbkdf2 -iter 100000`
- Ключ: `/etc/sheepfold/.usb_key` — 32 случайных байта из `/dev/urandom`
- Ключ **на роутере**, не на флешке — воткнуть флешку в компьютер и прочитать **невозможно**
- Включено по умолчанию (`encrypt=1`)

> **Важно:** при сбросе/замене роутера ключ теряется — архивы станут нерасшифруемы.
> UI предупреждает об этом перед форматированием и при включении флешки.

## UCI

```
config usb
    option device       '/dev/sda1'
    option enabled      '0'          # по умолчанию выключено
    option role         'logs_only'
    option encrypt      '1'
```

## API: `GET|POST /cgi-bin/sheepfold-api/usb-storage`

### GET

| `?action=` | Что возвращает |
|---|---|
| `scan` | Список USB/SD устройств: vendor, model, size_gb, role, current |
| `status` | Текущее состояние: подключено, занято, кол-во архивов |
| `info?dev=/dev/sda1` | Детали одного устройства |

### POST (JSON-тело)

| `action` | Что делает |
|---|---|
| `enable` | Сохраняет `device`, `role`, `encrypt` в UCI + mount |
| `disable` | umount + `enabled=0` |
| `format` | Быстрое форматирование |
| `setup-swap` | Создаёт и активирует swap-файл |
| `setup-extroot` | Прописывает fstab, нужна перезагрузка |
| `archive-push` | Ручной пуш архивов |

## Таб «Носитель USB» в LuCI — состав UI

1. **[Кнопка] Определить устройства** → `GET ?action=scan`
2. **[Список]** — `Самсунг 32ГБ (sda1)`, `Кингстон 16ГБ (sdb1)` ...
3. **[Чекбокс] Использовать выбранную флешку в системе** → `POST enable/disable`
   - Предупреждение при включении: «Для расширенных возможностей необходимы доп. пакеты»
4. **[Кнопка] Быстрое форматирование** → `POST format` (подтверждение: «все данные будут удалены!»)
5. **[Секция «Параметры»]** (collapse, по умолчанию свёрнут):
   - ☐ Файл подкачки (128 МБ) → `POST setup-swap`
   - ☐ Расширить место для пакетов (extroot) → `POST setup-extroot` + инфо о перезагрузке
   - ☑ Шифровать архивы логов AES-256 (по умолчанию ✔)
6. **[Status-блок]** — занято/свободно, кол-во архивов, роль

## Зависимости пакетов

| Функция | Пакет |
|---|---|
| Форматирование ext4 | `e2fsprogs`, `kmod-fs-ext4` |
| f2fs | `f2fs-tools`, `kmod-fs-f2fs` |
| Extroot | `block-mount` |
| Шифрование | `openssl-util` |
| USB | `kmod-usb-storage`, `kmod-scsi-generic` |

Все пакеты необязательны. UI предлагает установить нужные при включении соответствующей функции.
