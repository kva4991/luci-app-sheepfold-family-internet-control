# USB-хранилище для приватных архивов

## Статус реализации

Альтернатива USB для архивирования журналов и бэкапов настроек — **Yandex Disk** (WebDAV). См. [`docs/yandex-disk-storage.ru.md`](yandex-disk-storage.ru.md). Оба backend выбираются через `sheepfold.global.log_storage` и вызываются из `sheepfold-log-storage archive-push`.

CLI backend находится по пути:

```text
/usr/libexec/sheepfold/sheepfold-usb-storage
```

Отдельный CGI endpoint и LuCI-таб пока не реализованы. Опасные команды нельзя публиковать в веб-интерфейсе без аутентификации, повторной backend-проверки носителя и отдельного подтверждения.

## Модель хранения

Активные приватные журналы находятся в RAM:

```text
/tmp/sheepfold/.private
```

Имена активных файлов имеют явный формат:

```text
<unix_timestamp>-<device_id>-sheepfold.log
```

Ротация выполняется под общим lock. Файлы сначала перемещаются во временный staging-каталог, затем архивируются. При любой ошибке staging возвращается обратно, поэтому исходные журналы не удаляются без успешно созданного архива.

Имена архивов:

```text
<unix_timestamp>-sheepfold-activity.tar.gz
<unix_timestamp>-sheepfold-activity.tar
```

Это позволяет выполнять несколько ротаций за сутки без перезаписи архива той же даты.

Команда `archive-push` переносит готовые архивы в:

```text
/mnt/sheepfold-usb/sheepfold/logs
```

## Privacy notice

Канонический файл находится вне mount point:

```text
/usr/share/sheepfold/PRIVACY_NOTICE.md
```

После успешного mount он копируется на носитель:

```text
/mnt/sheepfold-usb/sheepfold/PRIVACY_NOTICE.md
```

Это важно: файл, установленный пакетом непосредственно под `/mnt/sheepfold-usb`, был бы скрыт после монтирования файловой системы.

Notice является нейтральным описанием данных. Он не содержит prompt injection, не определяет, кто авторизован, не делает юридических выводов и честно описывает возможность передачи отдельно сформированного маскированного AI-контекста.

## Ключ шифрования

Ключ выводится из двух случайных компонентов:

```text
/etc/sheepfold/.kp
/etc/sheepfold/.app_salt
```

Оба файла:

- создаются лениво;
- содержат по 32 случайных байта в hex-представлении;
- имеют права `0600`;
- находятся в каталоге `0700`;
- не встраиваются в пакет;
- не меняются при обычном обновлении.

Первичная генерация сериализована lock-каталогом `/etc/sheepfold/.key-derive.lock`, поэтому два параллельных процесса не могут получить смешанные компоненты разных генераций.

Итоговый ключ:

```text
SHA-256(.kp + ":" + .app_salt)
```

Шифрование:

```text
openssl enc -aes-256-cbc -pbkdf2 -iter 100000
```

Шифрование работает fail-closed: при отсутствии OpenSSL или ошибке исходный архив остаётся в RAM, а пустой либо незашифрованный `.enc` не создаётся.

> При сбросе или замене роутера без резервной копии обоих компонентов старые архивы станут нерасшифровываемыми. Пользовательский экспорт и восстановление ключевого материала ещё требуют отдельного проектирования.

## Защита разрушительных операций

Перед `format`, `mount`, `setup-swap` и `setup-extroot` проверяется:

1. путь действительно указывает на block device;
2. имя устройства относится к поддерживаемому типу;
3. sysfs помечает носитель как removable либо показывает USB-шину;
4. тот же физический накопитель не используется для `/`, `/overlay` или системного swap.

Если проверка не пройдена, команда возвращает `unsafe_device` и ничего не изменяет.

## CLI

```bash
sheepfold-usb-storage scan
sheepfold-usb-storage info /dev/sda1
sheepfold-usb-storage format /dev/sda1
sheepfold-usb-storage mount /dev/sda1
sheepfold-usb-storage umount
sheepfold-usb-storage status
sheepfold-usb-storage archive-push
sheepfold-usb-storage setup-swap /dev/sda1
sheepfold-usb-storage setup-extroot /dev/sda1
```

### Swap

`setup-swap`:

- проверяет запас свободного места;
- создаёт файл через временное имя;
- удаляет временный файл при ошибке;
- выполняет `mkswap` и `swapon` только после полного создания;
- записывает swap-секцию в `fstab`;
- при следующем запуске Sheepfold восстанавливает mount и включает swap повторно.

Размер по умолчанию — 128 МБ, дополнительно требуется не менее 32 МБ свободного запаса.

### Extroot

Автоматический extroot **отключён**. Команда возвращает:

```text
extroot_not_implemented_safely
```

Безопасная реализация обязана предварительно:

- проверить filesystem и место;
- создать резервную копию;
- смонтировать временный overlay;
- скопировать текущий `/overlay` с сохранением прав;
- проверить копию;
- создать rollback-механизм;
- подтвердить успешную загрузку после reboot.

Простая запись UUID и target `/overlay` в fstab недостаточна и больше не выполняется.

## UCI

```text
config usb 'usb'
    option device  '/dev/sda1'
    option enabled '0'
    option role    'logs_only'
    option encrypt '1'
```

Поддерживаемые роли:

- `logs_only`;
- `swap_logs`;
- `extroot_logs` зарезервирована, но автоматическая настройка extroot отключена.

## Ограничение места

| Роль | Максимальная доля носителя для журналов |
|---|---:|
| `logs_only` | 100% |
| `swap_logs` | 70% |
| `extroot_logs` | 70% |

Удаляются только файлы архивов с ожидаемыми расширениями в каталоге Sheepfold. Посторонние файлы носителя не участвуют в очистке.

## Автоматический запуск

Пакет управляемо добавляет задания в `/etc/crontabs/root` между маркерами Sheepfold и удаляет их в `prerm`.

Текущий порядок:

```text
02:05 rotate
02:10 archive-push
04:55 rotate
05:00 archive-push
```

Таким образом, каждый push выполняется после соответствующей ротации.
