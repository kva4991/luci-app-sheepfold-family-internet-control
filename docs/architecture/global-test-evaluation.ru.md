# Оценка ветки globalTest

Дата проверки: 2026-07-20.

Цель ветки — проверить на Sheepfold несколько лёгких практик из изученных наборов архитектурных и review-skills, не устанавливая их целиком и не добавляя тяжёлые runtime-зависимости.

## Что внедрено

1. Каталог ADR с шаблоном, индексом и десятью уже принятыми архитектурными решениями (§adrproc).
2. Root-cause процесс диагностики и правило свежего доказательства перед заявлением об исправлении (§debug01).
3. Советник влияния изменённых путей на LuCI, backend, UCI, Android, firewall, package и тестовые категории (§impact1).
4. Детерминированный all-pairs генератор для конфигураций формата пакета, редакции и интеграций (§pairmat).
5. Общие API- и UX-контракты, а также read-only browser-аудит доступных имён и размеров интерактивных элементов (§apicon1, §uxrev01).

Все новые генераторы используют только Node.js standard library. В production-пакет роутера, Android APK и LuCI не добавлена новая библиотека. `playwright-core@1.61.1` уже был закреплён существующим live-router harness и устанавливается только в ignored local test directory.

## Измеренная польза

- Матрица покрывает все допустимые пары факторов 15 сценариями вместо 384 комбинаций полного перебора.
- Проверка советника сразу обнаружила ложное совпадение слова `variant` в имени ADR с package build и заставила сузить правило до реальных путей.
- Модель runtime обнаружила неверный приоритет причины: `manualUnverified` мог скрыть более важный `missingAdguard`. Причина исправлена и закреплена тестом.
- Live LuCI прошёл 15 панелей в desktop `1440x900` и mobile `390x844`: нет horizontal overflow, безымянных команд, нулевых элементов, console errors и request failures.
- UX-аудит нашёл 41 неблокирующее предупреждение на mobile. Основные группы: сортируемые заголовки высотой 20 px, `Настроить`/редактирование/удаление 28x28 и `+30 мин` высотой 29 px. Отчёт теперь называет действие, а не только номер DOM-узла.

## Проверки

| Проверка | Результат |
|---|---|
| Новые ADR/impact/matrix и harness tests | 21/21 |
| `npm.cmd run lint:js` | успешно |
| `npm.cmd run test:smoke` | 262 успешно, 1 штатно пропущен |
| `npm.cmd run router:frontend` | `pass`, 41 неблокирующее UX-предупреждение |
| `npm.cmd test` вне sandbox | 404 успешно, 1 штатно пропущен, 657,6 секунды |

Первый полный прогон в ограниченной песочнице завершился только ошибками `spawnSync python EPERM` в IPK-тестах. Тот же неизменный набор вне sandbox прошёл полностью. Это подтверждает существующую инструкцию окружения и не является дефектом продукта.

## Стоимость и границы

- ADR требуют обновления только при изменении решения; подробности по-прежнему живут в focused-документах.
- Советник влияния является эвристикой. Неизвестный путь показывается явно; он не должен автоматически получать случайную категорию ради зелёного отчёта.
- All-pairs сокращает обычные сочетания, но не заменяет явные security, migration, rollback, malformed-input и live-router сценарии.
- UX-порог 32 px используется как неблокирующий ранний сигнал, а не как заявление о полном accessibility-аудите или соответствии конкретному стандарту.
- Browser smoke остаётся read-only и не проверяет сохранение форм, модалки, QR camera и Android UI.

## Вывод

Практики полезны и достаточно лёгкие для сохранения в проекте. Они уже нашли две ошибки в собственной инфраструктуре и один конкретный класс проблем текущего интерфейса. Устанавливать целиком VibeCoding, большие коллекции skills, LSP/MCP-обвязку или неприкреплённые AI-slop scanners не требуется: уникальная польза для Sheepfold не доказана, а контракты выше дают нужную часть поведения прозрачно и без vendor lock-in.

Мобильные hit areas стоит исправлять отдельной UI-правкой с визуальной проверкой, не смешивая её с внедрением тестовой инфраструктуры.
