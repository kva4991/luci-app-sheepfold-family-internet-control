# ADR-0007: Sheepfold выбирает AdGuard Home либо встроенную фильтрацию

- Статус: Принято
- Дата: 2026-07-20
- Теги: `§dompol`, `§aghplan`, `§slstres`, `§ipv6pod`
- Связанные документы: [Интеграции](../../integrations.md), [Политика доменов](../../site-domain-policy.ru.md)

## Контекст

Не на каждом роутере установлен AdGuard Home, но при его наличии дублирующие DNS-механизмы могут конфликтовать. Podkop отвечает за маршрутизацию и не должен становиться владельцем семейных списков сайтов.

## Рассмотренные варианты

1. Требовать AdGuard Home — исключает простые роутеры.
2. Всегда использовать встроенную фильтрацию — игнорирует уже установленный DNS-фильтр.
3. `auto|adguard|sheepfold` с безопасным fallback — выбранный вариант.

## Решение

Sheepfold остаётся владельцем семейной политики и выбирает исполнителя отдельно от топологии интеграций. В `auto` используется подтверждённый AdGuard Home, иначе встроенный backend. Автоуправление AdGuard Home ограничено одним точно принадлежащим Sheepfold URL-фильтром; пользовательские фильтры, YAML и Podkop не переписываются.

## Последствия

- Нерабочий источник или AdGuard не уничтожает последнюю рабочую политику.
- Статус различает «движок проверен» и «DNS-путь клиента проверен».
- Четыре топологии интеграций входят в тестовую матрицу.

## Проверка

`adguardIntegration.test.mjs`, `domainPolicyRuntime.test.mjs`, `siteListResilience.test.mjs` и реальные DNS-запросы LAN-клиента.

## Когда пересматривать

Если AdGuard Home изменит официальный API либо Podkop получит собственный проверяемый контракт семейной доменной политики.
