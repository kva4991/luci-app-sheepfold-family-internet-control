# ADR-0008: Обычный журнал живёт в RAM, экспорт маскирует данные

- Статус: Принято
- Дата: 2026-07-20
- Теги: `§b3nw8vp`, `§aimask1`
- Связанные документы: [Журнал](../../log-events.ru.md), [Приватность](../../privacy.ru.md)

## Контекст

Административный журнал нужен для расследования, но постоянная запись быстро изнашивает flash и экспорт может раскрыть домашние идентификаторы. Маскировка внутри локального интерфейса мешает владельцу понять событие.

## Рассмотренные варианты

1. Постоянный журнал на overlay — износ flash и накопление данных.
2. Маскировать всё при записи — ухудшает локальную диагностику.
3. RAM по умолчанию, полный локальный контекст и маскированный экспорт — выбранный вариант.

## Решение

Журнал по умолчанию хранится в `/tmp`, ограничивается сроком и размером и очищается после reboot. Локальные события показывают необходимые ID/MAC/IP, но никогда не содержат пароли, токены и ключи. Обычный экспорт маскирует чувствительные поля. Долгое USB/облачное хранение включается отдельно и не меняет default.

## Последствия

- Перезагрузка удаляет RAM-историю.
- Доставка уведомления и запись события являются разными операциями.
- Диагностический export не считается полной резервной копией.

## Проверка

`logStorage.test.mjs`, `logPanelUi.test.mjs` и ручная проверка маскированного экспорта.

## Когда пересматривать

Если OpenWrt предоставит стандартное wear-aware хранилище с понятными квотами и пользовательским контролем.
