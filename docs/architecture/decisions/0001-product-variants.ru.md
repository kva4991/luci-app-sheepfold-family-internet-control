# ADR-0001: Две редакции роутерного пакета и два общих Android APK

- Статус: Принято
- Дата: 2026-07-20
- Теги: `§prodvar`, `§owrtci1`
- Связанные документы: [Варианты продукта](../../product-variants.ru.md)

## Контекст

Контроль интернета должен работать без ИИ, а AI Support требует дополнительных backend-модулей и обработки данных. Разделение всего проекта на шесть независимых приложений привело бы к дрейфу одинаковой логики контроля.

## Рассмотренные варианты

1. Шесть независимых пакетов и APK — слишком много дублируемой навигации и бизнес-логики.
2. Один роутерный пакет со скрытым ИИ — Standard физически содержал бы ненужный AI-backend.
3. Две редакции роутерного пакета и два server-driven Android APK — выбранный вариант.

## Решение

Из общего кода собираются Standard и AI Support с одним внутренним package identity. Android выпускает только родительский и детский APK; AI UI открывается лишь после положительной capability роутера. Переход между редакциями сохраняет UCI-конфиг.

## Последствия

- Общий контроль устройств развивается один раз.
- Standard не содержит AI/activity backend.
- Сборка обязана проверять IPK/OpenWrt APK, обе редакции и отсутствие Android flavors.

## Проверка

`productVariants.test.mjs`, `openWrtVariantFeed.test.mjs` и `openWrtBuildWorkflow.test.mjs`. Локальные тесты не заменяют установку release-пакетов на поддерживаемых OpenWrt.

## Когда пересматривать

Если Android-редакции получат несовместимые пользовательские сценарии, которые нельзя безопасно закрыть capability-флагом.
