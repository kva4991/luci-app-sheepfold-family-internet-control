# Аварийно-полезные сайты: хранение и применение

<!-- §emerg1 -->

## Назначение

Аварийно-полезные сайты являются небольшим редактируемым исключением для устройств, интернет которых ограничен Sheepfold. Исключение не превращает устройство в участника белого списка устройств и не открывает административные интерфейсы роутера.

Каждая карточка хранится отдельной UCI-секцией `emergency_site`:

```uci
config emergency_site 'emergency_gosuslugi_ru'
        option domain 'gosuslugi.ru'
        option name 'Госуслуги'
        option description 'Государственные услуги'
        option enabled '1'
        option order '1'
```

LuCI принимает домен либо обычный URL, отбрасывает схему, путь, `www` и порт, затем проверяет ASCII/punycode-имя. IP-адреса, односоставные имена, дубликаты и строки с управляющими символами отклоняются. Добавление, редактирование и удаление остаются черновиком до общей кнопки `Сохранить настройки`.

## Применение трафика

`sheepfold-emergency-sites apply` выполняет четыре действия:

1. собирает и проверяет включённые UCI-домены;
2. при наличии поддержки `nftset` создаёт по одной служебной `dhcp`-секции `ipset` на домен;
3. разрешает базовые A/AAAA-адреса через `resolveip` либо безопасно разобранный ответ `nslookup`;
4. синхронизирует собственные nftables-наборы Sheepfold.

Один домен на одну DNS-секцию выбран намеренно. Слишком длинная общая строка `nftset=` способна помешать запуску dnsmasq; этот случай уже встречался в OpenWrt. См. [openwrt/openwrt #17548](https://github.com/openwrt/openwrt/issues/17548) и пример штатных `config ipset` в [openwrt/luci #8058](https://github.com/openwrt/luci/issues/8058).

Firewall разрешает только:

- TCP `80` и `443`;
- UDP `443` для QUIC;
- DNS и DHCP как необходимые локальные службы для устройств из чёрного списка устройств.

Совпавшее аварийное правило использует nftables `return`, а не безусловный `accept`. После него пакет всё равно проходит штатный firewall OpenWrt и маршрутизацию Podkop. Правила не меняют `meta mark`, `ct mark`, таблицы или маршруты Podkop.

## Чёрный список устройств

Устройство из чёрного списка устройств получает аварийный web-доступ только при `sheepfold.global.domain_allowlist_for_blocklist=1`. Независимо от этой настройки input guard продолжает запрещать LuCI, SSH, Sheepfold API и другие административные службы. Разрешение DNS/DHCP не является разрешением входа в интерфейс роутера.

## Совместимость DNS

- Если установлен dnsmasq с поддержкой `nftset`, ответы по домену и его поддоменам динамически попадают в IPv4/IPv6-наборы.
- Если прошивка собрана без `nftset`, Sheepfold не подсовывает dnsmasq неподдерживаемую директиву и использует резервное разрешение базового домена.
- При выбранном управляемом backend AdGuard Home аварийные исключения входят только в URL-фильтр `Sheepfold family site policy`; пользовательские фильтры и custom rules не переписываются. При ошибке API применяется встроенный dnsmasq/nftables fallback (§dompol).
- В режиме с Podkop те же наборы применяются до запрета Sheepfold, но без изменения правил маршрутизации Podkop.

## Ограничения модели

Доменное исключение на уровне роутера в итоге опирается на IP-адреса. Один IP CDN может обслуживать несколько сайтов, поэтому это не является криптографически строгой изоляцией ровно одного домена. Кроме того, современный сайт может загружать вход, карты, изображения или API с дополнительных технических доменов. Родителю следует добавлять только минимально необходимые зависимости и проверять результат отдельным ограниченным устройством.

Без DNS `nftset` гарантируется только резервная попытка открыть базовый домен; полноценность сложного сайта не обещается. Интерфейс не должен называть эту деградацию полной доменной фильтрацией.

## Проверка на живом роутере

1. Сохранить тестовый домен и убедиться, что появился `emergency_site`, а не только карточка в браузере.
2. Проверить `sheepfold-emergency-sites status`, nft-наборы и созданные `dhcp.sheepfold_emergency_*`.
3. Проверить обычное ограничение, расписание, глобальное отключение и чёрный список устройств отдельно.
4. Для чёрного списка устройств подтвердить доступ к разрешённому сайту и одновременный запрет LuCI, SSH и Sheepfold API.
5. Повторить тесты в четырёх режимах интеграции и после `fw4 reload`.
6. Удалить домен, сохранить настройки и проверить, что старые DNS-секции и адреса больше не дают исключение.
