# Сообщения и уведомления

Sheepfold должен поддерживать двухстороннее управление через мессенджер.

Поддерживаемые адаптеры:

- Telegram;
- VK;
- MAX, экспериментально.

На одном роутере одновременно может быть активен только один мессенджер.

## Правило активного мессенджера

- VK — вариант по умолчанию при первичной настройке мессенджера.
- Установленный конфиг должен держать мессенджер выключенным, пока не введены данные доступа и не привязан хотя бы один разрешённый администратор.
- На роутере может быть включён только один адаптер: `telegram`, `vk` или экспериментальный `max`.
- Переключение активного мессенджера должно требовать подтверждения администратора.
- Отключение текущего мессенджера не должно сразу удалять его настройки; их можно оставить для быстрого повторного включения, пока пользователь явно не удалит конфигурацию.
- Все действия мессенджера должны проходить через тот же Sheepfold API, который используют LuCI и Android.
- Опасные действия должны требовать явного подтверждения в выбранном мессенджере.

## Двухсторонний чат в Telegram

Telegram должен поддерживать и уведомления, и интерактивное управление родителем.

Планируемые события:

- включена глобальная блокировка;
- выключена глобальная блокировка;
- найдено новое неизвестное устройство;
- устройство добавлено в белый список;
- устройство добавлено в чёрный список;
- выдан временный доступ;
- сработало расписание;
- импорт/экспорт завершён;
- запрошена перезагрузка роутера;
- обновление приложения завершилось или упало;
- предупреждение интеграции с AdGuard Home;
- предупреждение интеграции с Podkop;
- ошибка firewall/rpcd/service;
- запрос ребёнка на доступ, если такой сценарий будет добавлен позже.

Интерактивные возможности:

- показать текущий статус;
- показать все устройства;
- искать устройства по имени, IP или MAC;
- включить/выключить глобальную блокировку с подтверждением;
- выдать временный доступ;
- добавить/удалить устройство из белого или чёрного списка;
- показать состояние Wi-Fi;
- включить Wi-Fi;
- выключить Wi-Fi;
- изменить автоматическое включение/выключение Wi-Fi;
- одобрить или отклонить запрос ребёнка на доступ;
- перед перезагрузкой, обновлением или импортом настроек задавать вопрос с подтверждением;
- принимать короткие естественные команды на русском языке.

## MAX Experimental

MAX можно оставить как экспериментальный адаптер.

Требования:

- не включать по умолчанию;
- не считать блокером первого стабильного релиза;
- использовать тот же внутренний интерфейс адаптеров;
- показывать в UI пометку `experimental`;
- не открывать роутер в интернет.

Архитектура:

- Telegram и VK реализовать отдельными адаптерами;
- код мессенджера не смешивать с firewall-логикой OpenWRT;
- все действия проводить через тот же Sheepfold API, который используют LuCI и Android;
- права команд явно ограничивать списком разрешённых пользователей.
- разрешённых администраторов назначать в настройках роутера;
- у каждого администратора должна быть роль и привязанный Telegram/VK/MAX user ID или chat ID;
- логировать административные действия: кто выполнил команду, что изменил, когда и с каким результатом.

## Реализация на OpenWRT

Бот должен работать как локальный сервис на роутере через `procd`.

Предпочтительная схема для Telegram:

- не открывать входящий порт на роутере;
- не требовать белый IP;
- использовать Telegram Bot API через исходящие HTTPS-запросы;
- получать сообщения через long polling `getUpdates`;
- отправлять ответы через HTTPS API;
- хранить offset последнего обработанного update локально;
- пропускать команды от неизвестных user ID / chat ID.

Эта схема подходит для домашнего роутера лучше, чем webhook, потому что webhook потребовал бы публичный HTTPS endpoint.

Для VK использовать тот же интерфейс адаптера. VK более обоснован как второй стабильный адаптер, чем MAX, потому что экосистема VK-ботов зрелее. MAX можно оставить экспериментальным.

Минимальный набор зависимостей для OpenWRT:

```text
uclient-fetch
ca-bundle
jsonfilter
```

Если позже shell-реализация станет слишком хрупкой, бот лучше перенести на `ucode`/`rpcd`-backend, но внешний API Sheepfold менять не нужно.

Полезные ссылки:

- Telegram Bot API: https://core.telegram.org/bots/api
- Telegram `getUpdates`: https://core.telegram.org/bots/api#getupdates
- VK Developers: https://dev.vk.com/

## Общие требования

- интеграция с мессенджером должна быть необязательной;
- токены ботов хранить безопасно;
- разрешить доверенные chat ID / user ID;
- не отправлять пароли роутера, API-токены, session ID, полные MAC-адреса, приватные заметки, полные списки устройств и другие чувствительные сетевые данные без явного разрешения;
- по умолчанию в сообщениях показывать понятное имя устройства и статус, а технические детали раскрывать только по запросу администратора;
- опасные действия должны требовать подтверждения: перезагрузка роутера, обновление, импорт настроек, очистка журнала, массовая блокировка, удаление правил, смена активного мессенджера, добавление нового администратора, включение/выключение всего Wi-Fi, изменение расписания Wi-Fi, изменение поведения WPS-кнопки;
- учитывать настройку журнала: если журнал отключён, живые уведомления можно отправлять, но локальную историю событий не писать без явного включения.
