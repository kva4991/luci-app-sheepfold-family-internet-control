# ИИ-помощник для родителей

В Android-приложении должна быть вкладка ИИ-помощника для родителей.

Помощник не управляет firewall сам. Он помогает родителю продумать семейные правила, расписания, постепенное развитие самоконтроля и спокойное общение без лишней эскалации.

## Архитектура провайдеров

Нужен абстрактный слой LLM-провайдеров.

Выбранная страна нахождения роутера должна определять, какие провайдеры нейросетей показываются в Android-приложении.

Требования:

- DeepSeek должен быть предпочтительным провайдером по умолчанию в странах, где он разрешён и доступен.
- Пользователь должен иметь возможность выбрать другого провайдера, если он разрешён в выбранной стране нахождения роутера.
- Нельзя зашивать одного глобального провайдера для всех стран.
- Нельзя показывать провайдеров, которые в профиле выбранной страны отмечены как недоступные, запрещённые или неподдерживаемые.
- Доступность провайдеров должна храниться в редактируемых country profiles, потому что юридическая и сетевая доступность может меняться.
- API-ключи хранить безопасно на Android, желательно через Android Keystore.
- Не хранить API-ключи провайдеров в UCI на OpenWRT, если позже явно не будет реализован server-side proxy mode.

## Передача данных ИИ

По умолчанию ИИ-помощнику отправляется только текст, который родитель сам написал в чате.

Запрещено автоматически отправлять ИИ-провайдеру:

- MAC-адреса;
- IP-адреса;
- имена детей;
- названия устройств;
- семейные детали;
- журнал действий;
- списки устройств;
- настройки роутера.

Если приложению нужно приложить контекст к вопросу, оно должно показать отдельное подтверждение:

```text
Отправить ИИ-провайдеру дополнительные сведения из Sheepfold? Это может включать названия устройств, правила доступа или семейный контекст.
```

В подтверждении должен быть виден список полей, которые будут отправлены. По умолчанию все чувствительные поля должны быть выключены.

## DeepSeek

DeepSeek — хороший кандидат на провайдера по умолчанию для родительского помощника, потому что он хорошо подходит для развёрнутых, рассуждающих и философских ответов.

Реализация должна использовать актуальную документацию DeepSeek API и актуальные имена моделей.

На этапе планирования DeepSeek документирует OpenAI-compatible API endpoint:

```text
https://api.deepseek.com
```

Имена моделей не нужно зашивать в код. Они должны храниться в конфигурации провайдера, чтобы устаревшие model ID можно было заменить без переделки архитектуры.

Промпт ИИ-помощника будет развиваться отдельно и перед production-релизом должен пройти ревью семейного психолога.

## Примеры запросов

- "Ребёнок спорит каждый раз, когда я выключаю интернет. Что делать?"
- "Помоги составить расписание для ребёнка 10 лет."
- "Как перейти от полного контроля к самоконтролю?"
- "Ребёнок обходит ограничения. Как реагировать без скандала?"
- "Предложи семейное соглашение про игры и уроки."

## Границы безопасности

- Помощник не должен тайно менять правила.
- Помощник не должен поощрять скрытую слежку.
- Помощник не должен предлагать наказание как первый вариант.
- Помощник не должен выдавать юридические, медицинские или психологические советы за профессиональную помощь.
