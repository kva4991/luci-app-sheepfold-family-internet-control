# Тест-кейсы Sheepfold

> Для AI-агентов: этот файл — обязательный чеклист перед любым PR. Читать вместе с `docs/backend-design.ru.md` и `docs/live-router-testing.ru.md`.

---

## Как использовать

- Каждый кейс имеет идентификатор формата `TC-XXX-NN`.
- Отметьте `[x]` перед отправкой PR.
- Кейсы разбиты на группы: LuCI-UI, API, Android, безопасность, граничные случаи.
- Кейсы «🔴 Критические» должны пройти обязательно; «🟡 Важные» — желательно.

---

## TC-UI: LuCI-интерфейс

### TC-UI-01 🔴 Root-пароль не задан
- **Сценарий**: Открываем Sheepfold в LuCI при пустом root-пароле.
- **Ожидается**: Настройки заблокированы, отображается предупреждение, ссылка на страницу пароля OpenWRT.

### TC-UI-02 🔴 Сохранение настроек только по кнопке
- **Сценарий**: Меняем несколько настроек (порт, тема, NTP). Не нажимаем «Сохранить».
- **Ожидается**: Изменения не применяются. Появляется подсказка «Есть несохранённые изменения».
- **После нажатия «Сохранить»**: все изменения применяются одним действием, LuCI-баннер «Не принятые изменения» не появляется.

### TC-UI-03 🔴 Вкладки «Списки пользователей»
- **Сценарий**: Открываем вкладку «Списки пользователей».
- **Ожидается**: Внутри — три вложенных вкладки: «Все устройства» (по умолчанию), «Белый список», «Чёрный список».

### TC-UI-04 🔴 Добавление устройства в оба списка одновременно
- **Сценарий**: Устройство в белом списке — пытаемся добавить его в чёрный.
- **Ожидается**: Интерфейс блокирует действие с понятным сообщением.

### TC-UI-05 🔴 Глобальная блокировка — подтверждение
- **Сценарий**: Нажимаем «Отключить интернет».
- **Ожидается**: Появляется диалог подтверждения. Без подтверждения интернет не отключается.

### TC-UI-06 🟡 Подтверждение опасных действий (блокировка, очистка журнала)
- **Сценарий**: Нажимаем «Очистить журнал», «Удалить группу», «Удалить сайт из аварийных».
- **Ожидается**: Каждое опасное действие требует подтверждения. Кнопка «Отмена» работает.

### TC-UI-07 🔴 Сохранение настроек — side effects
- **Сценарий**: Меняем порт API с 5201 на 5202 и сохраняем.
- **Ожидается**: UCI обновился, файл `/.well-known/sheepfold.json` содержит новый порт, сервис sheepfold перезапущен.

### TC-UI-08 🟡 WPS-предупреждение
- **Сценарий**: Выбираем «Подключить к Wi-Fi и добавить в белый список» для кнопки WPS.
- **Ожидается**: Показывается предупреждение об опасности. Подтверждение задержано на N секунд.

### TC-UI-09 🟡 Автоотключение Wi-Fi — предупреждение
- **Сценарий**: Устанавливаем автоотключение Wi-Fi в 23:00 и сохраняем.
- **Ожидается**: Показывается предупреждение «С телефона, подключённого по Wi-Fi, вы не сможете включить его обратно». Требуется подтверждение.

### TC-UI-10 🔴 Журнал — маскирование при экспорте
- **Сценарий**: В журнале есть записи с MAC-адресами и IP. Экспортируем журнал.
- **Ожидается**: В файле экспорта MAC-адреса вида `AA:BB:CC:xx:xx:FF`, IP вида `192.168.1.x`.

### TC-UI-11 🔴 Язык при установке English
- **Сценарий**: `install.sh` → `en` → установка IPK на чистый роутер.
- **Ожидается**: `sheepfold.global.language=en`, `luci.main.lang=en`, группы по умолчанию на английском, интерфейс Sheepfold на msgid (англ.). Автотест: `tests/installLanguage.test.mjs`.

## TC-API: Роутерный API

### TC-API-01 🔴 router-info не содержит секретов
- **Сценарий**: Запрашиваем `GET /cgi-bin/sheepfold-api/router-info`.
- **Ожидается**: В ответе нет полей: `password`, `token`, `key`, `mac` (клиентских), имён детей, списка устройств.

### TC-API-02 🔴 client-status — детский APK
- **Сценарий**: Запрашиваем `GET /cgi-bin/sheepfold-api/client-status` с IP устройства из DHCP.
- **Ожидается**: Ответ содержит `status`, `next_change_at`, `warning_before_minutes`.

### TC-API-03 🔴 Одноразовый код сопряжения
- **Сценарий**: Активируем код, используем его для сопряжения, пробуем использовать повторно.
- **Ожидается**: Повторное использование кода отклоняется с ошибкой.

### TC-API-04 🔴 Опасные команды требуют confirm-токен
- **Сценарий**: Отправляем команду `global-block-on` без confirm-токена.
- **Ожидается**: Ответ `{"error": "confirmation required", "code": 403}`.

### TC-API-05 🔴 Blocklist — наивысший приоритет
- **Сценарий**: Устройство одновременно в blocklist и allowlist (невозможно через UI, но проверяем через UCI вручную).
- **Ожидается**: Устройство заблокировано. Blocklist побеждает.

### TC-API-06 🟡 Диагностика — страновая цепочка
- **Сценарий**: Запрашиваем `router-info`, когда иностранные проверочные адреса недоступны, а российские адреса отвечают.
- **Ожидается**: `internet_status: online`, диагностика использует `ya.ru` / `gosuslugi.ru`.

### TC-API-07 🔴 Секреты не попадают в журнал
- **Сценарий**: Сохраняем токен Telegram, API-ключ DeepSeek. Читаем журнал.
- **Ожидается**: В журнале нет plaintext токенов и ключей.

## TC-AND: Android-приложение

### TC-AND-01 🔴 Первая настройка — порядок экранов
- **Сценарий**: Свежая установка APK, первый запуск.
- **Ожидается**: Соглашение → сеть → MAC-проверка → QR-сопряжение → PIN.

### TC-AND-02 🔴 Блокировка продолжения по мобильной сети
- **Сценарий**: Во время первой настройки отключаем Wi-Fi, включаем мобильные данные.
- **Ожидается**: Приложение блокирует продолжение с сообщением «Необходимо подключение к домашней сети».

### TC-AND-03 🔴 MAC-рандомизация
- **Сценарий**: На тестовом устройстве включена рандомизация MAC для тестовой сети.
- **Ожидается**: Приложение обнаруживает несовпадение MAC, показывает инструкцию по отключению рандомизации, не продолжает сопряжение до исправления.

### TC-AND-04 🔴 Предпросмотр диагностики перед отправкой AI
- **Сценарий**: Нажимаем «Отправить диагностику AI-ассистенту».
- **Ожидается**: Показывается полный предпросмотр данных, кнопка «Отправить» недоступна до подтверждения.

### TC-AND-05 🟡 Тема «следовать системе»
- **Сценарий**: Системная тема переключается со светлой на тёмную.
- **Ожидается**: Приложение автоматически меняет тему без перезапуска.

### TC-AND-06 🔴 Детский APK — нет административных функций
- **Сценарий**: Открываем детский APK.
- **Ожидается**: Нет кнопок управления устройствами, нет журнала, нет настроек, нет токенов администратора в хранилище.

### TC-AND-07 🟡 Уведомление за 5 минут до ограничения
- **Сценарий**: Детский APK, расписание заканчивается через 5 минут.
- **Ожидается**: Приложение показывает уведомление «Интернет ограничивается через 5 минут».

## TC-SEC: Безопасность

### TC-SEC-01 🔴 LuCI не строит shell-команды динамически
- **Проверка кода**: В `overview.js` и всех LuCI-файлах нет мест, где пользовательский ввод подставляется напрямую в `fs.exec()` без фиксированного списка аргументов.

### TC-SEC-02 🔴 API-ключи не в QR-коде
- **Сценарий**: Сканируем QR сопряжения.
- **Ожидается**: Payload вида `SF1|h=...|p=...|u=...|c=...`. Нет поля с токеном бота, AI-ключом, паролем root.

### TC-SEC-03 🔴 rpcd ACL — минимальный scope
- **Проверка конфига**: В ACL-файле Sheepfold перечислены только разрешённые пути. Нет wildcard `"*"` для exec.

### TC-SEC-04 🔴 Blocklist — нет доступа к LuCI/SSH/API
- **Сценарий**: Устройство в blocklist, пробуем открыть LuCI, SSH, Sheepfold API.
- **Ожидается**: Все три заблокированы. Доступны только «аварийно-полезные сайты» (если включено).

### TC-SEC-05 🟡 Экспорт без секретов по умолчанию
- **Сценарий**: Экспорт настроек в «читаемый JSON».
- **Ожидается**: Поля с ключевыми словами `password`, `token`, `secret`, `key` заменены на `[secret]`.

### TC-SEC-06 🔴 Bearer привязан к устройству
- **Сценарий**: Admin Bearer с другого телефона или без заголовков `X-Sheepfold-Device-Id` / `X-Sheepfold-Device-Mac`.
- **Ожидается**: `401 invalid_token`. Автотест: `tests/tokenDeviceBinding.test.mjs`.

## TC-EDGE: Граничные случаи

### TC-EDGE-01 🟡 Пересечение полуночи в расписании
- **Сценарий**: Расписание «разрешить с 22:00 до 02:00».
- **Ожидается**: В 23:00 доступ есть. В 01:30 доступ есть. В 02:01 доступ закрыт.

### TC-EDGE-02 🟡 Конфликт расписаний
- **Сценарий**: Два расписания для одной группы перекрываются.
- **Ожидается**: LuCI показывает предупреждение «Конфликт расписаний». Сохранение разрешено.

### TC-EDGE-03 🟡 Удаление группы с устройствами
- **Сценарий**: Пытаемся удалить группу, в которой есть устройства.
- **Ожидается**: Сообщение «Нельзя удалить группу с привязанными устройствами».

### TC-EDGE-04 🟡 Путь кэша не в /tmp/
- **Сценарий**: Вводим путь кэша `/var/log/sheepfold.log`.
- **Ожидается**: Ошибка «Путь должен начинаться с /tmp/».

### TC-EDGE-05 🔴 Порт вне диапазона
- **Сценарий**: Вводим порт `0` или `99999`.
- **Ожидается**: Ошибка «Введите порт от 1 до 65535».

### TC-EDGE-06 🟡 Журнал пуст после перезагрузки роутера
- **Сценарий**: Перезагружаем роутер, открываем журнал.
- **Ожидается**: «Журнал пуст». Предыдущие записи не сохранились в RAM. Если до перезагрузки был настроен Yandex Disk или USB, копии могли уйти на внешнее хранилище (см. [`docs/yandex-disk-storage.ru.md`](yandex-disk-storage.ru.md)).

### TC-STORAGE-YANDEX-01 🟡 Проверка входа и список файлов
- **Сценарий**: Настройки → Управление памятью → `Yandex Disk`, ввести логин и пароль приложения, сохранить. Нажать «Проверить вход», затем «Показать файлы на диске».
- **Ожидается**: Статус успеха; список `logs/` и `backups/` (или «Нет файлов» на пустом диске).

### TC-STORAGE-YANDEX-02 🟡 Статус синхронизации
- **Сценарий**: При активном `log_storage=yandex_disk` записать событие в журнал, подождать до 5 мин или вызвать `sheepfold-yandex-disk push-events` по SSH. Открыть панель Yandex Disk, нажать «Обновить статус синхронизации».
- **Ожидается**: Строка «Последняя синхронизация…» с временем и `ok:true` после успешной выгрузки.

### TC-STORAGE-YANDEX-03 🔴 Восстановление бэкапа с подтверждением
- **Сценарий**: На диске есть `sheepfold-config-*.tar.gz`. «Показать файлы» → выбрать бэкап → «Восстановить бэкап настроек» → подтвердить.
- **Ожидается**: Confirm перед действием; после успеха — перезагрузка страницы; `/etc/config/sheepfold` соответствует архиву; safety backup в `/tmp/sheepfold/`.

### TC-STORAGE-YANDEX-04 🔴 Секреты не в router-info и журнале
- **Сценарий**: Сохранить пароль Yandex Disk, вызвать `router-info`, прочитать журнал и экспорт.
- **Ожидается**: Пароль приложения нигде в открытом виде не фигурирует.

### TC-EDGE-07 🔴 Аварийно-полезные сайты — blocklist не открывает роутер
- **Сценарий**: Устройство в blocklist, включён доступ к аварийным сайтам.
- **Ожидается**: gosuslugi.ru открывается. LuCI, SSH, Sheepfold API недоступны.
