# Новые Wi-Fi-сети детского устройства

Функция помогает родителю увидеть, что телефон с явно установленным детским APK Sheepfold впервые подключился к новой Wi-Fi-сети. Она выключена по умолчанию и не участвует в решениях о доступе устройства в интернет (§childwifi1).

## Настройка

В LuCI: `Настройки` → `Уведомления` → `Уведомлять администраторов о новых Wi-Fi-сетях детского устройства`:

- `Да, с местоположением телефона при подключении` — уведомление содержит SSID и последнее доступное местоположение телефона;
- `Да, без местоположения` — уведомление содержит только SSID;
- `Нет (по умолчанию)` — детский APK не собирает и не отправляет отчёт о сети.

Отключение останавливает новые отчёты, но не удаляет уже сохранённые записи. Для удаления предназначена кнопка `Очистить сохранённую историю Wi-Fi-сетей` в той же вкладке.

## Поток данных

1. `/client-status` сообщает детскому APK, включён ли отчёт и разрешены ли координаты.
2. Только после разрешения APK проверяет активное Wi-Fi-подключение.
3. Телефон нормализует SSID и локально вычисляет SHA-256-отпечаток из `SSID + BSSID`.
4. Если домашний роутер недоступен, APK сохраняет до 100 подготовленных отчётов в закрытом хранилище приложения. При выключении функции очередь удаляется, а при запрете геолокации координаты вырезаются из ещё не отправленных записей.
5. После возвращения домой на роутер передаются SSID, необратимый отпечаток и, в разрешённом режиме, последнее доступное местоположение телефона. Открытый BSSID не передаётся.
6. Endpoint `/wifi-network-report` определяет устройство только по `REMOTE_ADDR` → DHCP/ARP. Переданные клиентом MAC или Sheepfold ID не принимаются.
7. Впервые увиденная сеть попадает в локальную историю, журнал и общую очередь администраторов. Родительский APK и настроенный семейный Telegram получают обычное административное уведомление.
8. Повторное подключение к тому же отпечатку не создаёт новую запись. На одно устройство хранится не больше 100 сетей.
9. Запись сети и находящиеся в той же строке координаты автоматически удаляются через 90 дней. После удаления старого отпечатка его повторное появление снова считается новой сетью.

История находится в `/etc/sheepfold/child-wifi-networks/`, имеет права `0700/0600`, сохраняется при обычном обновлении прошивки и доступна через аутентифицированный административный API `/child-wifi-networks`. Она не отправляется разработчику Sheepfold.

## Что означает местоположение

Координаты — это последняя доступная Android-позиция телефона не старше одного часа, а не проверенный адрес Wi-Fi-роутера. Телефон мог находиться на краю зоны покрытия, точка доступа могла перемещаться, а Android может не предоставить координаты либо сообщить их с большой погрешностью. Поэтому уведомление всегда называет это местоположением телефона при подключении и показывает точность.

Sheepfold не использует облачный канал для этой функции. Поэтому сеть вне дома попадёт к родителю только после возвращения детского телефона в сеть домашнего роутера. Фоновая проверка Android негарантированная: короткое подключение между двумя проверками может не попасть в список.

Для Android 9–12 чтение SSID/BSSID обычно требует разрешения местоположения. На новых Android дополнительно используется `NEARBY_WIFI_DEVICES`. Системный отказ в разрешении не ломает экран статуса: отчёт просто не создаётся.

## Границы доверия и законность

Данные APK являются уведомительным сигналом, а не доказательством местонахождения ребёнка: модифицированное приложение способно отправить вымышленные данные. Функция не должна использоваться для автоматического наказания или как единственный источник решения о безопасности.

Пользователь отвечает за законные полномочия, возраст ребёнка, необходимые согласия и допустимость обработки геолокации по применимому праву. Разрешение операционной системы само по себе не заменяет такое правовое основание.

## Проверка

Контракт защищает `tests/childWifiNetworkMonitoring.test.mjs`. На физическом телефоне дополнительно проверяются: отказ в разрешениях, неизвестный SSID, отсутствие свежей геопозиции, повторное подключение, смена точки доступа с тем же SSID, доставка нескольким родительским телефонам и очистка истории.
