# Конфигурация Android-приложения Sheepfold

> Этот документ описывает, как Android-клиент хранит настройки, как происходит сопряжение с роутером и какие параметры передаются по API.
> Для AI-агентов: читайте этот файл вместе с `docs/backend-design.ru.md` и `docs/android-openwrt-api.ru.md`.

---

## Общая архитектура хранения настроек

Android-приложение **не хранит** никаких семейных правил, устройств или UCI-настроек локально.
Все данные живут на роутере. Приложение — только интерфейс поверх роутерного API.

| Что хранится на Android | Что хранится на роутере |
|---|---|
| Адрес роутера (IP / hostname) | Все UCI-настройки Sheepfold |
| Порт API (по умолчанию 5201) | Токены аутентификации (хэши) |
| Логин текущего администратора | Списки устройств, групп, расписаний |
| Кэш последнего снимка роутера | API-ключи DeepSeek / Gemini |
| Тема оформления (системная / светлая / тёмная) | Токен мессенджера |
| Язык приложения | — |
| Флаг «показано ли соглашение» | — |

## Первоначальная настройка (First Setup)

### Порядок экранов при первом запуске

1. **Соглашение** — чекбокс «Я принимаю пользовательское соглашение…» обязателен до продолжения.
2. **Подключение к домашней сети** — требуется Wi-Fi или Ethernet до роутера; продолжение по мобильной сети заблокировано.
3. **Проверка MAC-адреса Wi-Fi** — если для этой сети включена рандомизация MAC, показывается инструкция и требуется подтверждение после переключения на реальный MAC. Если подключение по Ethernet — предупреждение о видимом MAC, инструкция Wi-Fi пропускается.
4. **Сопряжение с роутером** — сканирование QR-кода из LuCI или ручной ввод (адрес, логин, код сопряжения).
5. **Локальный PIN / пароль приложения** — рекомендуется PIN или пароль; биометрика предлагается как дополнительный вариант с предупреждением.

### QR-код сопряжения

Текущий формат payload: `SF2|h=<адрес>|p=<порт>|u=<логин>|c=<код>|spki=<SHA-256>`

- `h` — IP-адрес или hostname роутера
- `p` — порт Sheepfold API
- `u` — логин администратора
- `c` — одноразовый код (10 символов, генерирует роутер)
- `spki` — SHA-256 отпечаток публичного TLS-ключа роутера

Tokен действует **10 минут** и после успешного сопряжения сразу аннулируется бэкендом.
Ответ `/pair` ещё не сохраняется как готовое подключение: APK выполняет обычный авторизованный запрос к роутеру и открывает главное окно только после подтверждения, что токен связан с устройством администратора в основном UCI-конфиге (§pairtx1).

## Обнаружение роутера в сети

<!-- §dscqr01 -->

Приложение сначала проверяет discovery-файл через штатный HTTPS-порт роутера:

```
GET https://<router-ip>/.well-known/sheepfold.json
```

Если штатный HTTPS недоступен, дополнительно проверяется `https://<router-ip>:5201/.well-known/sheepfold.json`. Из ответа приложение узнаёт фактический `appPort`/`httpsPort`, после чего обязательно проверяет `GET /cgi-bin/sheepfold-api/ping`. У discovery свой счётчик запросов: исчерпанный лимит обычных API-команд не должен выглядеть как отсутствие сервера. Статический JSON без работающего API не считается найденным сервером.

Ожидаемый ответ:

```json
{
  "service": "sheepfold",
  "name": "Sheepfold Family Internet Control",
  "routerName": "OpenWRT Sheepfold",
  "appPort": "5201",
  "apiPath": "/cgi-bin/sheepfold-api",
  "apiBase": "/cgi-bin/sheepfold-api",
  "version": "0.1.0"
}
```

HTTP fallback не используется: автопоиск и сопряжение остаются только на HTTPS.

## Тема оформления

- Значение по умолчанию: **«следовать системе»** (`system`).
- Поддерживаемые варианты: `system`, `light`, `dark`.
- Хранится локально в `SharedPreferences` / DataStore.
- Тема не отправляется на роутер и не влияет на LuCI.

## Детский APK (Child APK)

Отдельная сборка без авторизации и управляющих функций:

- Не требует входа / пароля при открытии.
- Показывает только статус текущего устройства: разрешён / ограничен / расписание.
- Кнопка «Попросить +30 минут» — отправляет запрос родителю через API роутера, не применяет самостоятельно.
- Не имеет доступа к спискам устройств, настройкам, журналу.
- Не хранит никаких токенов администратора.
- При первом запуске приложение до 30 секунд само ищет Sheepfold на шлюзе активной Wi-Fi/Ethernet-сети и проверяет `/ping` по HTTPS.
- Только если поиск не удался, показываются подсказка о возможной неверной Wi-Fi-сети, повторный поиск и ручной ввод IP-адреса. Ручной ввод не является обычным первым экраном.
- MAC и идентификатор не вводятся в APK: роутер определяет устройство по фактическому локальному соединению.

Endpoint для детского APK:
```
GET /cgi-bin/sheepfold-api/client-status
```

Ответ:
```json
{
  "status": "allowed",         // allowed | restricted | scheduled
  "schedule_name": "Учебные дни",
  "next_change_at": "20:30",
  "warning_before_minutes": 5
}
```

> Уведомление отображается за **5 минут** до ограничения (поле `warning_before_minutes`).

## AI-ассистент в Android

### Провайдеры

| Провайдер | Хранение ключа | Кто делает запрос |
|---|---|---|
| DeepSeek | На роутере | Роутер |
| Gemini Free | На роутере | Роутер |

Android **не хранит** API-ключи. Все запросы к AI проходят через роутерный прокси.

### Отправка диагностического контекста

1. Android запрашивает снимок: `GET /cgi-bin/sheepfold-api/router-info`
2. Показывает родителю предпросмотр перед отправкой.
3. Родитель подтверждает отправку.
4. Приложение отправляет данные выбранному провайдеру через роутер.

В снимке **отсутствуют**: пароли Wi-Fi, токены ботов, имена детей, MAC-адреса, списки устройств.

## Уведомления

| Событие | Канал | Задержка |
|---|---|---|
| Новое устройство в сети | Push (если настроен мессенджер) / локальное уведомление | немедленно |
| Временный доступ заканчивается | Локальное + мессенджер | за 5 мин |
| Роутер недоступен > 2 мин | Локальное | через 2 мин |
| Обновление доступно | Локальное | при открытии / фоновая проверка |
| В детском телефоне обнаружена или изменена SIM-карта | Общая очередь родительских APK + активный мессенджер по настройке | при очередном опросе детского приложения |
