# Источники белых и чёрных списков сайтов

Sheepfold не должен вручную поддерживать большой список “безопасных для детей” сайтов внутри репозитория. Такие списки быстро устаревают, содержат ошибки и требуют отдельной редакционной ответственности.

Правильная модель: Sheepfold хранит настройки источников, скачивает обновляемые внешние списки на роутере, нормализует домены, кэширует результат и показывает родителю понятный режим применения.

## Белые списки

Белый список источников нужен для строгого режима группы: “открывать только явно разрешённые домены”.

Стартовый источник:

- UT1 `child` — категория сайтов, разрешённых для детей младше 10 лет: https://dsi.ut-capitole.fr/blacklists/index_en.php

Важно: UT1 прямо описывает свою базу как категоризацию, а не готовую команду “всё это блокировать или разрешать”. Поэтому Sheepfold должен показывать источник как подсказку/базу, а не как гарантию абсолютной безопасности.

Родитель должен иметь возможность добавить свои источники в формате:

```text
Название источника | URL
```

В `Настройки -> Разное -> Источники списков сайтов` должна быть настройка `Обновление списков сайтов`:

- `каждый день`;
- `раз в 3 дня`;
- `раз в неделю` — значение по умолчанию.

Эта настройка управляет фоновым обновлением кэша списков на роутере. Обновление не должно выполняться прямо из LuCI-страницы, чтобы интерфейс не зависал на скачивании больших файлов.

## Чёрные списки

Чёрный список сайтов нужен для запрета опасных/нежелательных категорий для обычных устройств. Значения режима:

- `не включать`;
- `включить для всех`;
- `включить для всех кроме белого списка и админов`.

Стартовые обновляемые источники:

- UT1 категории `adult`, `malware`, `phishing`, `gambling`, `games`, `vpn`, `dangerous_material`, `drogue`, `redirector`: https://dsi.ut-capitole.fr/blacklists/index_en.php
- StevenBlack hosts с расширениями gambling/porn/social/fakenews: https://github.com/StevenBlack/hosts
- HaGeZi DNS blocklists, особенно Threat Intelligence Feeds и списки обхода фильтрации: https://github.com/hagezi/dns-blocklists
- URLhaus для вредоносных URL/доменов: https://urlhaus.abuse.ch/api/

## Как применять

Backend на роутере должен:

- скачивать списки отдельным процессом, не из LuCI-страницы;
- поддерживать разные форматы: plain domains, hosts, Adblock/DNS, архивы категорий;
- нормализовать домены к нижнему регистру;
- удалять дубли из итоговых списков на backend-стороне, как это делает podkop при обработке списков;
- удалять комментарии, пустые строки, `0.0.0.0`, `127.0.0.1`;
- хранить кэш в `/tmp/sheepfold/site-lists/`, если возможно;
- показывать дату последнего успешного обновления и размер кэша;
- не ломать работу AdGuard Home: если AdGuard Home включён, Sheepfold должен передавать/использовать списки совместимо, а не дублировать тяжёлую DNS-фильтрацию без причины;
- при ошибке обновления продолжать использовать последний рабочий кэш и показать понятное предупреждение.

## Группы

В настройках группы есть режим:

```text
Разрешить только выбранные источники белых списков для этой группы
```

Если режим включён, устройства группы должны получать доступ только к:

- доменам из выбранных источников белых списков;
- вручную добавленным аварийно-полезным сайтам, если это разрешено правилами;
- технически необходимым доменам, без которых выбранный сайт не работает, только после явного подтверждения родителя или через понятную диагностику.

Белый список устройств и группа `Без ограничений` остаются отдельными механизмами. Чёрный список устройств всегда выше любых сайт-списков.
