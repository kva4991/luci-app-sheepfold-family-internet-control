# Профили роутеров Sheepfold

Каталог содержит только настройки, которые нельзя безопасно реализовать общим OpenWrt-механизмом.
Имя каталога профиля должно точно совпадать с `board_name`, например:

```text
cudy,wr3000s-v1/
```

## Управление LED

Общая логика Sheepfold работает через `/sys/class/leds` и используется для любого роутера, у которого нет файла:

```text
<board_name>/led/apply-mode
```

Имя `apply-mode` означает «применить выбранный пользователем режим LED». Оно намеренно отличается от `fixLed/install`: режим применяется повторно при запуске, а исправление изменяет постоянную конфигурацию OpenWrt.

Если `led/apply-mode` существует, он становится профильным backend для этой модели. Скрипт обязан самостоятельно вызвать общий backend для тех LED, которые не требуют специальных команд:

```sh
/usr/libexec/sheepfold/sheepfold-router-control-legacy led-apply
```

Не следует копировать специальные регистры, MDIO-команды или имена устройств между моделями без подтверждения по DTS и теста на оборудовании.

Необязательная зависимость объявляется двумя текстовыми файлами:

```text
<board_name>/led/dependency-package
<board_name>/led/dependency-command
```

Первый файл содержит единственное имя пакета `opkg`, второй — команду, которая должна появиться после установки. LuCI показывает пользователю имя пакета и устанавливает его только после явного подтверждения.

## Обратимые исправления LED

Исправление, которое меняет постоянную конфигурацию OpenWrt, хранится отдельно:

```text
<board_name>/fixLed/install
<board_name>/fixLed/uninstall
```

Требования к скриптам:

- операции должны быть идемпотентными;
- `install` сначала проверяет, не реализована ли эквивалентная настройка пользователем или OpenWrt;
- созданные UCI-секции должны иметь уникальное имя и признак владения Sheepfold;
- `uninstall` удаляет только секции с признаком владения Sheepfold;
- пользовательские и системные секции нельзя изменять или удалять;
- при невозможности безопасной установки скрипт завершается с ошибкой;
- изменение должно применяться штатным init-скриптом OpenWrt;
- отключение галочки в LuCI должно полностью откатывать только изменение Sheepfold.

Если каталог `fixLed` для модели отсутствует, галочка ремонта в LuCI блокируется, а управление LED продолжает использовать общий backend.

## Cudy WR3000S v1

Профиль `cudy,wr3000s-v1` содержит:

- MDIO-команду для отключения аппаратных LED коммутатора;
- необязательную зависимость `mdio-tools`;
- исправление WAN Internet LED через отдельную UCI-секцию `system.sheepfold_wan_status`;
- безопасное удаление этой секции при отключении исправления.
